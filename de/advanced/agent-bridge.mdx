## Übersicht

Agent Bridge ermöglicht es AI-Agenten (Claude Code, Cursor, Cowork usw.), die Web2MD Chrome-Erweiterung fernzusteuern, um URLs im Batch zu konvertieren — insbesondere Reddit, JS-gerenderte und login-geschützte Seiten, die serverseitigen API-Zugriff blockieren.

Statt die Web2MD API aufzurufen (die Reddit blockiert), sendet der AI-Agent Befehle über [Native Messaging](https://developer.chrome.com/docs/extensions/develop/concepts/native-messaging) an Ihre lokale Chrome-Erweiterung. Die Erweiterung öffnet jede Seite in einem Hintergrund-Tab unter Verwendung **Ihrer echten Browser-Sitzung** — mit Ihren Cookies und Ihrem Login-Status —, extrahiert den Inhalt, konvertiert ihn in Markdown und liefert das Ergebnis zurück.

<Note>
  Agent Bridge erfordert einen **PRO-Plan** und den auf Ihrem Rechner installierten **Native Messaging Host**. Die Erweiterung muss in Chrome geöffnet sein.
</Note>

## Architektur

```
AI Agent ←MCP/stdio→ MCP Server ←TCP:12315→ Native Host ←NM→ Chrome Extension → Any Website
```

| Komponente | Rolle |
|-----------|------|
| **AI Agent** | Claude Code, Cursor, Cowork — ruft MCP-Tools auf |
| **MCP Server** | Übersetzt MCP-Tool-Aufrufe in TCP-Nachrichten |
| **Native Host** | Vermittelt zwischen TCP und Chrome Native Messaging |
| **Chrome Extension** | Öffnet Tabs, extrahiert HTML, konvertiert in Markdown |

Die gesamte Kommunikation erfolgt **lokal** — nichts verlässt Ihren Rechner. Der Native Host lauscht ausschließlich auf `localhost:12315`.

## Einrichtung

### Schritt 1: MCP Server bauen

```bash
cd packages/mcp-server
pnpm build
```

### Schritt 2: Native Messaging Host installieren

```bash
cd packages/mcp-server
./install.sh <your-extension-id>
```

<Tip>
  Finden Sie Ihre Extension-ID unter `chrome://extensions` bei aktiviertem Entwicklermodus. Suchen Sie nach Web2MD und kopieren Sie die ID-Zeichenkette (z. B. `ijmgpkkfgpijifldbjafjiapehppcbcn`).
</Tip>

<Warning>
  Nach der Installation müssen Sie Chrome **vollständig beenden (Cmd+Q auf dem Mac) und neu öffnen**. Chrome liest Native-Messaging-Manifeste nur beim Start — ein bloßes Neuladen der Erweiterung genügt nicht.
</Warning>

Das Installationsskript:
- Kopiert die Host-Dateien nach `~/.web2md/` (umgeht macOS-TCC-Beschränkungen für `~/Desktop`)
- Ermittelt den absoluten Pfad zu `node` (Chrome startet mit minimalem PATH)
- Schreibt das NM-Manifest in Chromes `NativeMessagingHosts`-Verzeichnis

### Schritt 4: Überprüfen

Öffnen Sie in Chrome `chrome://extensions` → klicken Sie auf den "Service Worker"-Link von Web2MD → prüfen Sie die Konsole:

```
[Web2MD] Service worker started
[Web2MD] Connected to native host
[Web2MD] Native host TCP relay ready on port 12315
```

Wenn Sie diese drei Zeilen sehen, funktioniert Agent Bridge.

## Verfügbare Tools

Agent Bridge stellt drei MCP-Tools bereit — wählen Sie je nach Transportweg und Batching-Bedarf. Alle drei sind vollständig auf der Seite [MCP Server](/de/advanced/mcp-server) dokumentiert; hier folgt der kurze Vergleich.

| Tool | Transport | Batching |
|------|-----------|----------|
| `bridge_convert_url` | HTTP (Erweiterung ruft HTML ab, Server konvertiert es) | Einzelne URL |
| `agent_convert` | Native Messaging (Architektur dieser Seite) | Einzelne URL, optionaler `skill`-Parameter |
| `agent_batch_convert` | Native Messaging | Bis zu 50 URLs, sequenziell |

### agent_convert

Konvertiert eine einzelne URL über die Chrome-Erweiterung per Native Messaging.

| Parameter | Typ | Beschreibung |
|-----------|------|-------------|
| `url` | string | Die zu konvertierende URL |
| `skill` | string (optional) | Skill-ID oder -Name — dessen Inhalt wird dem zurückgegebenen Markdown angehängt |

**Rückgabe:** Markdown-Inhalt mit Titel, Quell-URL, Wortanzahl und Lesezeit.

**Am besten geeignet für:** Reddit-Threads, login-geschützte Seiten, JS-gerenderte Seiten.

### agent_batch_convert

Konvertiert bis zu 50 URLs im Batch. Die URLs werden sequenziell verarbeitet — die Erweiterung öffnet jede Seite in einem Hintergrund-Tab, extrahiert den Inhalt, schließt den Tab und geht dann zur nächsten über.

| Parameter | Typ | Beschreibung |
|-----------|------|-------------|
| `urls` | string[] | Array der zu konvertierenden URLs (max. 50) |

**Rückgabe:** Pro-URL-Ergebnisse, gestreamt sobald sie fertig sind, plus eine Zusammenfassung.

**Am besten geeignet für:** Research-Workflows — Reddit-Threads, HN-Diskussionen oder Konkurrenzseiten im Batch für die AI-Analyse konvertieren.

### bridge_convert_url

Konvertiert eine einzelne URL, indem ihr HTML über die Erweiterung per lokaler HTTP-Bridge abgerufen wird (statt per Native Messaging), und konvertiert dann serverseitig. Für Einzelkonvertierungen funktional ähnlich zu `agent_convert`, erfordert aber nicht den Native Messaging Host — es muss lediglich die Erweiterung installiert und aktiv sein.

| Parameter | Typ | Beschreibung |
|-----------|------|-------------|
| `url` | string | Die zu konvertierende URL |
| `extensionId` | string (optional) | Überschreibt die Extension-ID (wird bei Weglassen automatisch anhand Ihres API-Schlüssels ermittelt) |

<Info>
  Alle erfolgreichen Konvertierungen werden automatisch in Ihrer [Dashboard History](https://web2md.org/dashboard/history) gespeichert, mit AI-generierten Zusammenfassungen und Tags.
</Info>

Jede URL ohne dedizierten Extraktor greift auf die vollständige HTML-Extraktion der Seite zurück — sie funktioniert weiterhin, nur ohne die seitenspezifische Bereinigung (Entfernen von Navigations-Chrome, Werbung, Kommentar-Widgets usw.), die die spezialisierten Extraktoren vornehmen. Seiten mit einer öffentlichen API (Reddit, Hacker News, GitHub, Stack Overflow) werden nach Möglichkeit über diese API abgerufen — schneller und zuverlässiger als das Öffnen eines Hintergrund-Tabs. Alles andere öffnet die Seite in einem Hintergrund-Tab und extrahiert aus dem gerenderten DOM.

## Fehlerbehebung

<AccordionGroup>
  <Accordion title="'Native host not installed' oder 'not found'">
    Chrome hat das NM-Manifest nicht geladen. **Beenden Sie Chrome vollständig (Cmd+Q) und öffnen Sie es neu.** Ein bloßes Neuladen der Erweiterung genügt nicht.
  </Accordion>
  <Accordion title="'Native host has exited' sofort">
    Chrome kann das Host-Skript nicht ausführen. Häufige Ursachen:
    - `node` nicht gefunden — führen Sie `./install.sh` erneut aus, das den absoluten Node-Pfad verwendet
    - Der Host befindet sich in einem TCC-geschützten Verzeichnis (`~/Desktop`, `~/Documents`) — führen Sie die Installation erneut aus, um ihn nach `~/.web2md/` zu verschieben
  </Accordion>
  <Accordion title="'Agent connection error' vom MCP">
    Der TCP-Server des Native Host läuft nicht. Stellen Sie sicher, dass:
    1. Chrome geöffnet ist
    2. Die Web2MD-Erweiterung geladen ist
    3. Die Service-Worker-Konsole "TCP relay ready on port 12315" anzeigt
  </Accordion>
  <Accordion title="'Failed to extract content from the page'">
    Die Erweiterung ist nicht angemeldet. Öffnen Sie das Web2MD-Popup in Chrome und melden Sie sich bei Ihrem PRO-Konto an.
  </Accordion>
  <Accordion title="'Tab load timeout'">
    Die Zielseite braucht zu lange zum Laden. Das ist bei langsamen Seiten normal. Die Erweiterung wartet bis zu 15 Sekunden pro Tab. Reddit nutzt den JSON-API-Extraktor und benötigt keinen Tab — Timeouts bei Reddit bedeuten daher meist, dass die Post-URL ungültig ist (404).
  </Accordion>
  <Accordion title="Konvertierungen erscheinen nicht in der Dashboard History">
    Das Speichern in der History erfordert, dass die Erweiterung mit einem PRO-Konto angemeldet ist. Der Speichervorgang ist Fire-and-Forget — schlägt der API-Aufruf stillschweigend fehl, erscheinen die Konvertierungen nicht. Prüfen Sie, ob Ihr Auth-Token gültig ist.
  </Accordion>
</AccordionGroup>

## Unterschied zum MCP Server

| Funktion | `convert_url` | `bridge_convert_url` | `agent_convert` / `agent_batch_convert` |
|---------|--------------|----------------------|------------------------------------------|
| Wo es läuft | Serverseitiger API-Aufruf | Erweiterung ruft HTML ab, Server konvertiert es | Ihr lokaler Chrome-Browser (Native Messaging) |
| Reddit-Unterstützung | ❌ Von Reddit blockiert | ✅ Nutzt echte Browser-Sitzung | ✅ Nutzt echte Browser-Sitzung |
| Login-geschützte Seiten | ❌ Kein Zugriff | ✅ Nutzt Ihre Cookies | ✅ Nutzt Ihre Cookies |
| JS-gerenderte Seiten | ❌ Kein JavaScript | ✅ Vollständiges Chrome-Rendering | ✅ Vollständiges Chrome-Rendering |
| Erforderliche Einrichtung | Nur API-Schlüssel | Erweiterung installiert + aktiv | Erweiterung + Native Messaging Host installiert |
| Batching | — | Nur einzelne URL | Bis zu 50 URLs (`agent_batch_convert`) |
| Chrome muss geöffnet sein | Nein | Ja | Ja |

**Faustregel:** Verwenden Sie `convert_url` für öffentliche Seiten. Verwenden Sie `bridge_convert_url`, wenn Sie die Browser-Sitzung der Erweiterung nutzen möchten, aber den Native Messaging Host nicht installieren wollen. Verwenden Sie `agent_convert` / `agent_batch_convert` für Reddit, authentifizierte Seiten, JS-lastige Seiten, oder wenn Sie im Batch arbeiten müssen — oder einen Skill über den `skill`-Parameter von `agent_convert` anwenden möchten.
