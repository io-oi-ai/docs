## Übersicht

Mit der Web2MD CLI konvertieren Sie jede URL direkt von Ihrem Terminal aus in sauberes Markdown. Leiten Sie die Ausgabe per Pipe an LLMs weiter, verarbeiten Sie URL-Listen im Batch, oder übernehmen Sie Inhalte in Ihren Obsidian-Vault — alles ohne einen Browser zu öffnen.

```bash
npx web2md https://example.com/article
```

<Note>
  Die CLI erfordert **Node.js 18+**. Führen Sie `node -v` aus, um Ihre Version zu prüfen.
</Note>

## Installation

Keine Installation nötig — einfach mit `npx` ausführen:

```bash
npx web2md <url> [options]
```

Oder global installieren für schnelleren Start:

```bash
npm install -g web2md-cli
web2md <url> [options]
```

## Modi

Die Web2MD CLI arbeitet je nach Konfiguration in drei Modi:

<CardGroup cols={3}>
  <Card title="Local" icon="laptop">
    **Standardmodus.** Kein API-Schlüssel erforderlich. Ruft Seiten ab und konvertiert lokal. Funktioniert für die meisten öffentlichen Websites.
  </Card>
  <Card title="Server" icon="server">
    **Mit API-Schlüssel.** Setzen Sie `WEB2MD_API_KEY`, um Reddit, Fandom/Wikia und andere eingeschränkte Seiten freizuschalten, die serverseitige Verarbeitung erfordern.
  </Card>
  <Card title="Bridge" icon="bridge">
    **Mit dem Flag `--bridge`.** Nutzt Ihre Chrome-Erweiterung, um JS-gerenderte oder login-geschützte Seiten abzurufen, die statisches Fetching nicht erreichen kann.
  </Card>
</CardGroup>

## Flags

| Flag | Beschreibung |
|------|-------------|
| `--no-images` | Entfernt Bildverweise aus der Ausgabe |
| `--no-links` | Entfernt Hyperlinks aus der Ausgabe |
| `--meta` | Fügt YAML-Frontmatter hinzu (title, source, wordCount, tokenCount, readingTime, date) |
| `--json` | Gibt als JSON `{ markdown, metadata }` aus |
| `-o, --output <file>` | Schreibt die Ausgabe in eine Datei |
| `--output-dir <dir>` | Schreibt jede URL in eine separate `.md`-Datei |
| `--batch <file>` | Liest URLs aus einer Datei (eine pro Zeile, `#` = Kommentar) |
| `--vault <dir>` | Obsidian-Vault-Modus: speichert in `<dir>/raw/` und aktualisiert `<dir>/INDEX.md` |
| `--concurrency <n>` | Maximale parallele Abrufe (Standard: 3, max: 20) |
| `--bridge` | Nutzt die Chrome-Erweiterung für JS-gerenderte oder login-geschützte Seiten |
| `-q, --quiet` | Unterdrückt Fortschrittsmeldungen |

## Befehle

Neben der Konvertierung von URLs bietet die CLI zwei Subbefehle für Authentifizierung und Library-Sync:

| Befehl | Beschreibung |
|---------|-------------|
| `web2md login` | Einmalig authentifizieren und den API-Schlüssel in `~/.web2md/config.json` speichern |
| `web2md sync [dir]` | Synchronisiert Ihre Web2MD Library als Markdown-Dateien in ein lokales Verzeichnis |

### login

```bash
npx web2md-cli login
```

Startet einen browserbasierten Device-Authorization-Flow: Die CLI öffnet `https://web2md.org` und zeigt einen Code zur Bestätigung an. Nach der Bestätigung wird Ihr API-Schlüssel in `~/.web2md/config.json` gespeichert — Sie müssen `WEB2MD_API_KEY` nicht mehr in jeder Shell exportieren. Der [MCP-Server](/de/advanced/mcp-server#authentifizierung) liest aus derselben Datei, sodass eine einmalige Anmeldung hier auch MCP-Tool-Aufrufe authentifiziert.

Möchten Sie stattdessen einen Schlüssel direkt einfügen, statt den Browser-Flow zu nutzen?

```bash
npx web2md-cli login --key
```

Dies fragt auf stdin nach einem Schlüssel (`w2m_...`) und speichert ihn auf dieselbe Weise. Holen Sie sich einen Schlüssel im [Dashboard](https://web2md.org/dashboard/api-keys).

### sync

```bash
npx web2md-cli sync [dir] [--tag <t>] [--full]
```

Synchronisiert Ihre Library — jede Seite, die Sie über die Erweiterung, das Dashboard oder die API gespeichert haben — als Markdown-Dateien mit Frontmatter in ein lokales Verzeichnis, organisiert nach Tag. Erfordert eine Anmeldung (oder gesetztes `WEB2MD_API_KEY`).

| Flag | Beschreibung |
|------|-------------|
| `dir` | Zielverzeichnis (Standard: `./web2md-vault`) |
| `--tag <t>` | Synchronisiert nur Einträge mit diesem Tag |
| `--full` | Ignoriert den inkrementellen Cursor und scannt den gesamten Verlauf neu |

Die Synchronisierung ist standardmäßig inkrementell: Sie verfolgt einen Cursor in `<dir>/.web2md-sync.json` und ruft bei nachfolgenden Läufen nur neue Einträge ab. Richten Sie Obsidian (oder ein beliebiges Markdown-Tool) auf das Zielverzeichnis, um Ihre Library als Notizen zu durchsuchen.

```bash
npx web2md-cli login
npx web2md-cli sync ~/Documents/Web2MD-Library
```

## Umgebungsvariablen

| Variable | Beschreibung |
|----------|-------------|
| `WEB2MD_API_KEY` | API-Schlüssel (`w2m_xxx`) für Reddit, eingeschränkte Seiten und `sync` — hat Vorrang vor `~/.web2md/config.json` |
| `WEB2MD_API_URL` | Überschreibt die Basis-URL der API |
| `WEB2MD_EXTENSION_ID` | Überschreibt die Chrome-Extension-ID für den `--bridge`-Modus |

<Tip>
  Fügen Sie diese Ihrem Shell-Profil hinzu (`~/.zshrc` oder `~/.bashrc`), damit sie sitzungsübergreifend bestehen bleiben:
  ```bash
  export WEB2MD_API_KEY="w2m_your_key_here"
  ```
</Tip>

## Anwendungsbeispiele

### Einfache Konvertierung

```bash
npx web2md https://example.com/article
```

Gibt Markdown auf stdout aus.

### Per Pipe an ein LLM

```bash
npx web2md https://react.dev/learn/thinking-in-react | llm "Summarize this page"
```

```bash
npx web2md https://docs.python.org/3/tutorial/classes.html | claude "Explain the key concepts"
```

### In Datei speichern

```bash
npx web2md https://example.com/article -o article.md
```

```bash
npx web2md https://example.com/article --meta -o article.md
```

Das Flag `--meta` stellt YAML-Frontmatter mit Titel, Quell-URL, Wortanzahl, Token-Anzahl, Lesezeit und Datum voran.

### Batch aus Datei

Erstellen Sie eine Datei `urls.txt`:

```text
# Research papers
https://arxiv.org/abs/2301.00001
https://arxiv.org/abs/2301.00002

# Blog posts
https://example.com/blog/post-1
https://example.com/blog/post-2
```

Führen Sie dann aus:

```bash
npx web2md --batch urls.txt --output-dir ./research --concurrency 5
```

Jede URL wird als separate `.md`-Datei im Verzeichnis `./research` gespeichert.

### Obsidian-Vault-Übernahme

```bash
npx web2md --batch urls.txt --vault ~/Documents/MyVault
```

Dies speichert jede Seite unter `~/Documents/MyVault/raw/` und aktualisiert `~/Documents/MyVault/INDEX.md` mit Links zu allen konvertierten Seiten.

### Reddit mit API-Schlüssel

```bash
export WEB2MD_API_KEY="w2m_your_key_here"
npx web2md https://www.reddit.com/r/LocalLLaMA/comments/example
```

<Warning>
  Reddit erfordert einen gültigen API-Schlüssel. Ohne diesen schlagen Reddit-URLs aufgrund der Bot-Beschränkungen von Reddit fehl.
</Warning>

### Bridge-Modus

Nutzen Sie die Chrome-Erweiterung, um JS-gerenderte oder login-geschützte Seiten zu verarbeiten:

```bash
npx web2md --bridge https://app.example.com/dashboard
```

<Note>
  Der Bridge-Modus erfordert eine installierte Web2MD Chrome-Erweiterung und ein laufendes Chrome. Die CLI kommuniziert über Chromes Native-Messaging-Protokoll mit der Erweiterung.
</Note>

### JSON-Ausgabe

```bash
npx web2md --json https://example.com/article
```

Liefert eine strukturierte Ausgabe:

```json
{
  "markdown": "# Article Title\n\nContent here...",
  "metadata": {
    "title": "Article Title",
    "source": "https://example.com/article",
    "wordCount": 1250,
    "tokenCount": 1680,
    "readingTime": 5,
    "date": "2026-04-11T10:30:00.000Z"
  }
}
```

Nützlich für die programmatische Weiterverarbeitung oder zum Weiterleiten an `jq`:

```bash
npx web2md --json https://example.com/article | jq '.metadata.tokenCount'
```

## Optimierte Seiten

Web2MD enthält integrierte Adapter für diese Seiten, die eine sauberere Ausgabe als die generische Konvertierung liefern:

<AccordionGroup>
  <Accordion title="Seiten mit optimierter Unterstützung">
    - **Wikipedia** — saubere Artikelextraktion, Infobox-Behandlung
    - **arXiv** — Paper-Abstracts und Metadaten
    - **Hacker News** — Threads mit Kommentaren
    - **GitHub** — Issues und Pull Requests
    - **Stack Overflow** — Fragen und Antworten
    - **dev.to** — Blogbeiträge
    - **Medium** — Artikel (umgeht die Paywall-Vorschau)
    - **Substack** — Newsletter-Beiträge
    - **OpenAI Docs** — Dokumentationsseiten
    - **Mintlify-basierte Docs** — auf Mintlify aufgebaute Dokumentations-Websites
    - **Reddit** — Beiträge und Kommentare (erfordert API-Schlüssel)
  </Accordion>
</AccordionGroup>

## Gängige Workflows

<AccordionGroup>
  <Accordion title="Dokumentation an einen AI-Agenten übergeben">
    ```bash
    npx web2md --batch docs-urls.txt --output-dir ./context --quiet
    ```
    Richten Sie das Kontextverzeichnis Ihres AI-Agenten auf `./context`, um fundierte Antworten zu erhalten.
  </Accordion>
  <Accordion title="Einen Research-Korpus aufbauen">
    ```bash
    npx web2md --batch papers.txt --vault ~/Obsidian/Research --meta --concurrency 10
    ```
    Erstellt einen indizierten, durchsuchbaren Research-Vault in Obsidian.
  </Accordion>
  <Accordion title="Formatierung für LLM-Eingaben entfernen">
    ```bash
    npx web2md --no-images --no-links https://example.com/article | llm "Analyze this"
    ```
    Entfernt Bilder und Links, um den Token-Verbrauch beim Weiterleiten an LLMs zu reduzieren.
  </Accordion>
</AccordionGroup>
