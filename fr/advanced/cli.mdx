## Vue d'ensemble

Le CLI Web2MD vous permet de convertir n'importe quelle URL en Markdown propre directement depuis votre terminal. Envoyez le résultat à des LLM, traitez des listes d'URL en lot, ou ingérez du contenu dans votre coffre Obsidian — le tout sans ouvrir de navigateur.

```bash
npx web2md https://example.com/article
```

<Note>
  Le CLI nécessite **Node.js 18+**. Exécutez `node -v` pour vérifier votre version.
</Note>

## Installation

Aucune installation nécessaire — exécutez simplement avec `npx` :

```bash
npx web2md <url> [options]
```

Ou installez-le globalement pour un démarrage plus rapide :

```bash
npm install -g web2md-cli
web2md <url> [options]
```

## Modes

Le CLI Web2MD fonctionne selon trois modes en fonction de votre configuration :

<CardGroup cols={3}>
  <Card title="Local" icon="laptop">
    **Mode par défaut.** Aucune clé API requise. Récupère et convertit les pages en local. Fonctionne pour la plupart des sites web publics.
  </Card>
  <Card title="Serveur" icon="server">
    **Avec clé API.** Définissez `WEB2MD_API_KEY` pour débloquer Reddit, Fandom/Wikia et d'autres sites restreints nécessitant un traitement côté serveur.
  </Card>
  <Card title="Bridge" icon="bridge">
    **Avec l'option `--bridge`.** Utilise votre extension Chrome pour récupérer des pages rendues en JS ou protégées par connexion, inaccessibles à la récupération statique.
  </Card>
</CardGroup>

## Options

| Option | Description |
|------|-------------|
| `--no-images` | Supprime les références d'images du résultat |
| `--no-links` | Supprime les hyperliens du résultat |
| `--meta` | Ajoute un en-tête YAML (title, source, wordCount, tokenCount, readingTime, date) |
| `--json` | Sortie au format JSON `{ markdown, metadata }` |
| `-o, --output <file>` | Écrit le résultat dans un fichier |
| `--output-dir <dir>` | Écrit chaque URL dans un fichier `.md` séparé |
| `--batch <file>` | Lit les URL depuis un fichier (une par ligne, `#` = commentaire) |
| `--vault <dir>` | Mode coffre Obsidian : enregistre dans `<dir>/raw/` et met à jour `<dir>/INDEX.md` |
| `--concurrency <n>` | Nombre max de récupérations parallèles (défaut : 3, max : 20) |
| `--bridge` | Utilise l'extension Chrome pour les sites rendus en JS ou protégés par connexion |
| `-q, --quiet` | Supprime les messages de progression |

## Commandes

Au-delà de la conversion d'URL, le CLI propose deux sous-commandes pour l'authentification et la synchronisation de la Bibliothèque :

| Commande | Description |
|---------|-------------|
| `web2md login` | S'authentifie une fois et enregistre votre clé API dans `~/.web2md/config.json` |
| `web2md sync [dir]` | Synchronise votre Bibliothèque Web2MD vers un répertoire local sous forme de fichiers Markdown |

### login

```bash
npx web2md-cli login
```

Démarre un flux d'autorisation par appareil basé sur le navigateur : le CLI ouvre `https://web2md.org` et affiche un code à approuver. Une fois approuvé, votre clé API est enregistrée dans `~/.web2md/config.json` — plus besoin d'exporter `WEB2MD_API_KEY` dans chaque shell. Le [serveur MCP](/fr/advanced/mcp-server#authentication) lit depuis le même fichier, donc se connecter une fois ici authentifie aussi les appels d'outils MCP.

Vous préférez coller directement une clé plutôt que passer par le flux navigateur ?

```bash
npx web2md-cli login --key
```

Cela demande une clé (`w2m_...`) sur l'entrée standard et l'enregistre de la même manière. Obtenez une clé depuis le [tableau de bord](https://web2md.org/dashboard/api-keys).

### sync

```bash
npx web2md-cli sync [dir] [--tag <t>] [--full]
```

Synchronise votre Bibliothèque — chaque page enregistrée via l'extension, le tableau de bord ou l'API — vers un répertoire local sous forme de fichiers Markdown avec en-tête YAML, organisés par tag. Nécessite d'être connecté (ou que `WEB2MD_API_KEY` soit défini).

| Option | Description |
|------|-------------|
| `dir` | Répertoire cible (défaut : `./web2md-vault`) |
| `--tag <t>` | Synchronise uniquement les éléments avec ce tag |
| `--full` | Ignore le curseur incrémental et rescanne l'historique complet |

La synchronisation est incrémentale par défaut : elle suit un curseur dans `<dir>/.web2md-sync.json` et ne récupère que les nouveaux éléments lors des exécutions suivantes. Pointez Obsidian (ou tout autre outil lisant du Markdown) vers le répertoire cible pour parcourir votre Bibliothèque sous forme de notes.

```bash
npx web2md-cli login
npx web2md-cli sync ~/Documents/Web2MD-Library
```

## Variables d'environnement

| Variable | Description |
|----------|-------------|
| `WEB2MD_API_KEY` | Clé API (`w2m_xxx`) pour Reddit, les sites restreints, et `sync` — prioritaire sur `~/.web2md/config.json` |
| `WEB2MD_API_URL` | Remplace l'URL de base de l'API |
| `WEB2MD_EXTENSION_ID` | Remplace l'ID de l'extension Chrome pour le mode `--bridge` |

<Tip>
  Ajoutez-les à votre profil shell (`~/.zshrc` ou `~/.bashrc`) pour qu'elles persistent entre les sessions :
  ```bash
  export WEB2MD_API_KEY="w2m_your_key_here"
  ```
</Tip>

## Exemples d'utilisation

### Conversion basique

```bash
npx web2md https://example.com/article
```

Affiche le Markdown sur la sortie standard.

### Envoyer à un LLM

```bash
npx web2md https://react.dev/learn/thinking-in-react | llm "Summarize this page"
```

```bash
npx web2md https://docs.python.org/3/tutorial/classes.html | claude "Explain the key concepts"
```

### Enregistrer dans un fichier

```bash
npx web2md https://example.com/article -o article.md
```

```bash
npx web2md https://example.com/article --meta -o article.md
```

L'option `--meta` ajoute un en-tête YAML avec le titre, l'URL source, le nombre de mots, le nombre de tokens, le temps de lecture et la date.

### Traitement en lot depuis un fichier

Créez un fichier `urls.txt` :

```text
# Research papers
https://arxiv.org/abs/2301.00001
https://arxiv.org/abs/2301.00002

# Blog posts
https://example.com/blog/post-1
https://example.com/blog/post-2
```

Puis exécutez :

```bash
npx web2md --batch urls.txt --output-dir ./research --concurrency 5
```

Chaque URL est enregistrée dans un fichier `.md` séparé dans le répertoire `./research`.

### Ingestion dans un coffre Obsidian

```bash
npx web2md --batch urls.txt --vault ~/Documents/MyVault
```

Cela enregistre chaque page dans `~/Documents/MyVault/raw/` et met à jour `~/Documents/MyVault/INDEX.md` avec des liens vers toutes les pages converties.

### Reddit avec clé API

```bash
export WEB2MD_API_KEY="w2m_your_key_here"
npx web2md https://www.reddit.com/r/LocalLLaMA/comments/example
```

<Warning>
  Reddit nécessite une clé API valide. Sans clé, les URL Reddit échoueront en raison des restrictions anti-bot de Reddit.
</Warning>

### Mode Bridge

Utilisez l'extension Chrome pour gérer les pages rendues en JS ou protégées par connexion :

```bash
npx web2md --bridge https://app.example.com/dashboard
```

<Note>
  Le mode Bridge nécessite que l'extension Chrome Web2MD soit installée et que Chrome soit en cours d'exécution. Le CLI communique avec l'extension via le protocole de messagerie native de Chrome.
</Note>

### Sortie JSON

```bash
npx web2md --json https://example.com/article
```

Renvoie une sortie structurée :

```json
{
  "markdown": "# Article Title\n\nContent here...",
  "metadata": {
    "title": "Article Title",
    "source": "https://example.com/article",
    "wordCount": 1250,
    "tokenCount": 1680,
    "readingTime": 5,
    "date": "2026-04-11T10:30:00.000Z"
  }
}
```

Utile pour une consommation programmatique ou pour envoyer à `jq` :

```bash
npx web2md --json https://example.com/article | jq '.metadata.tokenCount'
```

## Sites optimisés

Web2MD inclut des adaptateurs intégrés pour ces sites, produisant un résultat plus propre que la conversion générique :

<AccordionGroup>
  <Accordion title="Sites avec prise en charge optimisée">
    - **Wikipedia** — extraction propre des articles, gestion des infobox
    - **arXiv** — résumés et métadonnées d'articles
    - **Hacker News** — fils de discussion avec commentaires
    - **GitHub** — Issues et Pull Requests
    - **Stack Overflow** — questions et réponses
    - **dev.to** — articles de blog
    - **Medium** — articles (contourne l'aperçu du paywall)
    - **Substack** — articles de newsletter
    - **OpenAI Docs** — pages de documentation
    - **Docs basées sur Mintlify** — sites de documentation construits sur Mintlify
    - **Reddit** — posts et commentaires (nécessite une clé API)
  </Accordion>
</AccordionGroup>

## Workflows courants

<AccordionGroup>
  <Accordion title="Alimenter un agent IA en documentation">
    ```bash
    npx web2md --batch docs-urls.txt --output-dir ./context --quiet
    ```
    Pointez le répertoire de contexte de votre agent IA vers `./context` pour des réponses ancrées.
  </Accordion>
  <Accordion title="Construire un corpus de recherche">
    ```bash
    npx web2md --batch papers.txt --vault ~/Obsidian/Research --meta --concurrency 10
    ```
    Crée un coffre de recherche indexé et consultable dans Obsidian.
  </Accordion>
  <Accordion title="Supprimer la mise en forme pour une entrée LLM">
    ```bash
    npx web2md --no-images --no-links https://example.com/article | llm "Analyze this"
    ```
    Supprime les images et les liens pour réduire la consommation de tokens lors de l'envoi à un LLM.
  </Accordion>
</AccordionGroup>
