## Обзор

MCP-сервер Web2MD предоставляет Web2MD в виде набора инструментов, которые AI-ассистенты могут вызывать напрямую, используя [Model Context Protocol (MCP)](https://modelcontextprotocol.io) — открытый стандарт для подключения AI-моделей к внешним инструментам и источникам данных.

После настройки вы можете попросить Claude, Cursor или Windsurf конвертировать страницы, искать и синхронизировать вашу Library, выполнять batch-конвертацию через расширение Chrome или подключить сохранённый Skill — не покидая диалог.

<Note>
  Для работы MCP-сервера требуется **API-ключ Web2MD** (тариф PRO). Сгенерируйте его в [дашборде](https://web2md.org/dashboard/api-keys) или выполните `npx web2md-cli login`, чтобы авторизоваться один раз и автоматически передать ключ CLI (см. [Аутентификацию](#authentication) ниже).
</Note>

## Доступные инструменты

MCP-сервер предоставляет 11 инструментов, сгруппированных по назначению:

| Инструмент | Описание |
|------|-------------|
| `convert_url` | Конвертирует любой URL в чистый Markdown через серверный конвертер |
| `bridge_convert_url` | Конвертирует URL через ваше расширение Chrome по HTTP — для Reddit, страниц с JS-рендерингом или защищённых логином |
| `agent_convert` | Конвертирует URL через расширение Chrome по native messaging (те же сценарии, что и у `bridge_convert_url`, но другой транспорт) |
| `agent_batch_convert` | Batch-конвертация до 50 URL через расширение Chrome по native messaging |
| `search_library` | Поиск по сохранённой Library по ключевому слову и/или тегу |
| `get_library_items` | Получение полного Markdown для элементов Library по id |
| `sync_library` | Синхронизация вашей Library с локальной директорией в виде Markdown-файлов |
| `list_skills` | Список Skills, сохранённых в вашем облачном аккаунте Web2MD |
| `get_skill` | Получение полного содержимого одного Skill по id или имени |
| `semantic_search` | **Устарел** — используйте вместо него `search_library` + `get_library_items` |
| `get_conversion` | **Устарел** — используйте вместо него `search_library` + `get_library_items` |

### convert_url

Получает URL на стороне сервера и возвращает содержимое страницы в виде Markdown с заголовком, количеством слов и временем чтения. Поддерживает те же опции конвертации, что и REST API (`includeImages`, `includeLinks`).

**Пример запроса:** _"Конвертируй этот URL в markdown: https://docs.example.com/getting-started"_

### bridge_convert_url

Конвертирует URL, получая его HTML через запущенное расширение Chrome (по локальному HTTP-мосту), а затем конвертируя этот HTML в Markdown на стороне сервера. Используйте для страниц, к которым сервер не может обратиться напрямую — Reddit, сайты за логином или страницы с активным JS-рендерингом. Требует открытого Chrome с установленным расширением Web2MD; ID расширения определяется автоматически по вашему API-ключу.

**Пример запроса:** _"Используй bridge, чтобы конвертировать этот тред на Reddit: https://reddit.com/r/..."_

### agent_convert

Конвертирует URL так же, как `bridge_convert_url` — через ваше расширение Chrome — но по каналу [Agent Bridge](/ru/advanced/agent-bridge) через native messaging вместо HTTP. Принимает необязательный параметр `skill`: передайте id или имя Skill, и его содержимое будет добавлено к возвращённому Markdown, чтобы агент мог сразу его применить. Требует запущенного Chrome с установленным расширением и native host.

**Пример запроса:** _"Конвертируй эту страницу с применением skill summarize: https://..."_

### agent_batch_convert

Тот же транспорт, что и у `agent_convert`, но принимает массив из до 50 URL и конвертирует их последовательно за один вызов. Лучше всего подходит для получения множества тредов Reddit или страниц с JS-рендерингом за один раз.

**Пример запроса:** _"Batch-конвертируй эти 12 URL в Markdown: [...]"_

### search_library

Ищет по вашей Library Web2MD — по всем сохранённым страницам — по ключевому слову (проверяется заголовок и полное содержимое страницы) и/или тегу, от новых к старым. Возвращает компактные метаданные (id, title, url, date, tags), а не полный Markdown. Аккаунты на бесплатном тарифе видят в результатах только самые последние элементы; инструмент сообщает, если результаты были усечены.

**Пример запроса:** _"Найди в моей библиотеке всё об аутентификации"_

### get_library_items

Получает полное содержимое Markdown для элементов Library по id (до 20 за вызов — сначала используйте `search_library`, чтобы найти id). Ответы могут быть большими; для длинных страниц запрашивайте не более 5 id за раз.

**Пример запроса:** _"Получи полное содержимое элементов библиотеки abc123 и def456"_

### sync_library

Синхронизирует всю вашу Library с локальной директорией в виде Markdown-файлов, организованных по тегам с frontmatter. По умолчанию инкрементальная (отслеживает курсор в `<dir>/.web2md-sync.json`); передайте `full: true`, чтобы пересканировать всё заново. Это та же синхронизация, что запускает `web2md sync` из [CLI](/ru/advanced/cli) — используйте тот интерфейс, который подходит вашему workflow.

**Пример запроса:** _"Синхронизируй мою библиотеку в ~/notes/web2md, только тег 'research'"_

### list_skills

Выводит список Skills, сохранённых в вашем облачном аккаунте Web2MD, в виде компактного массива (id, name, description). Используйте `get_skill`, чтобы получить один Skill полностью.

**Пример запроса:** _"Какие skills у меня сохранены?"_

### get_skill

Получает полное содержимое одного Skill в стиле SKILL.md по id или имени — его инструкции, описание и содержимое целиком. Сначала используйте `list_skills`, если не знаете id.

**Пример запроса:** _"Получи мой skill 'summarize'"_

### semantic_search / get_conversion (устарели)

Эти два инструмента появились раньше Library. Они всё ещё работают, но видят только преобразования, сохранённые через старый history API, а не всю Library — новым интеграциям следует использовать `search_library` и `get_library_items`.

## Аутентификация

Сервер определяет API-ключ в следующем порядке:

1. Переменная окружения `WEB2MD_API_KEY`, если она задана.
2. Иначе — `~/.web2md/config.json`, тот же файл, который создаёт `npx web2md-cli login` (device-login flow [CLI](/ru/advanced/cli)). Выполните `login` один раз, и и CLI, и MCP-сервер автоматически подхватят ключ — блок `env` не нужен.

Если ключ не найден ни там, ни там, сервер запустится, но каждый вызов инструмента будет возвращать ошибку аутентификации с подсказкой задать ключ или выполнить `login`.

## Настройка

<Tabs>
  <Tab title="Claude Desktop">
    Откройте файл конфигурации Claude Desktop:

    - **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
    - **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

    Добавьте запись `web2md` в секцию `mcpServers`:

    ```json
    {
      "mcpServers": {
        "web2md": {
          "command": "npx",
          "args": ["-y", "web2md-mcp-server"],
          "env": {
            "WEB2MD_API_KEY": "w2m_your_key_here"
          }
        }
      }
    }
    ```

    Если вы уже выполнили `npx web2md-cli login`, блок `env` можно полностью убрать — сервер прочитает ключ из `~/.web2md/config.json`.

    Перезапустите Claude Desktop. В списке инструментов должны появиться инструменты Web2MD.
  </Tab>
  <Tab title="Cursor">
    Откройте файл конфигурации MCP для Cursor по пути `~/.cursor/mcp.json` и добавьте:

    ```json
    {
      "mcpServers": {
        "web2md": {
          "command": "npx",
          "args": ["-y", "web2md-mcp-server"],
          "env": {
            "WEB2MD_API_KEY": "w2m_your_key_here"
          }
        }
      }
    }
    ```

    Перезапустите Cursor, чтобы загрузить новый сервер.
  </Tab>
  <Tab title="Windsurf">
    Откройте файл конфигурации MCP для Windsurf по пути `~/.windsurf/mcp.json` и добавьте:

    ```json
    {
      "mcpServers": {
        "web2md": {
          "command": "npx",
          "args": ["-y", "web2md-mcp-server"],
          "env": {
            "WEB2MD_API_KEY": "w2m_your_key_here"
          }
        }
      }
    }
    ```

    Перезапустите Windsurf, чтобы загрузить новый сервер.
  </Tab>
</Tabs>

## Примеры использования

После запуска MCP-сервера вы можете использовать естественный язык в вашем AI-ассистенте:

<AccordionGroup>
  <Accordion title="Конвертировать страницу в Markdown">
    **Вы:** Конвертируй этот URL в markdown: https://react.dev/learn/thinking-in-react

    **Ассистент:** _(вызывает `convert_url`)_ Вот конвертированный Markdown для "Thinking in React"...
  </Accordion>
  <Accordion title="Поиск и получение из вашей Library">
    **Вы:** Найди в моей библиотеке всё об аутентификации, затем получи полное содержимое самого последнего элемента

    **Ассистент:** _(вызывает `search_library`, затем `get_library_items`)_ Я нашёл 3 элемента, связанных с аутентификацией. Вот полное содержимое самого последнего — "OAuth 2.0 Guide"...
  </Accordion>
  <Accordion title="Синхронизировать вашу Library локально">
    **Вы:** Синхронизируй мою библиотеку в ~/notes/web2md

    **Ассистент:** _(вызывает `sync_library`)_ Синхронизировано 42 новых элемента, 118 уже актуальны, 0 ошибок.
  </Accordion>
  <Accordion title="Batch-конвертация через расширение">
    **Вы:** Batch-конвертируй эти 8 тредов Reddit: [...]

    **Ассистент:** _(вызывает `agent_batch_convert`)_ Успешно конвертировано 8/8 URL.
  </Accordion>
  <Accordion title="Применить сохранённый Skill">
    **Вы:** Какие у меня есть skills, и можешь применить "summarize" к этой странице: https://...

    **Ассистент:** _(вызывает `list_skills`, затем `agent_convert` с `skill: "summarize"`)_ Вот страница, конвертированная в Markdown, с применённым ниже skill summarize...
  </Accordion>
</AccordionGroup>

<Tip>
  Комбинируйте инструменты в рамках одного диалога. Например: _"Найди в моей библиотеке документацию по React, затем получи полное содержимое самого последнего элемента"_.
</Tip>

## Устранение неполадок

<AccordionGroup>
  <Accordion title="Инструменты не появляются после настройки">
    Убедитесь, что вы перезапустили приложение после редактирования файла конфигурации. Проверьте, что `npx -y web2md-mcp-server` успешно запускается в вашем терминале.
  </Accordion>
  <Accordion title="Ошибки аутентификации">
    Убедитесь, что `WEB2MD_API_KEY` — это корректный ключ `w2m_`, или что он есть в `~/.web2md/config.json` (если не уверены, повторно выполните `npx web2md-cli login`). Проверить ключ можно на [странице API-ключей](https://web2md.org/dashboard/api-keys).
  </Accordion>
  <Accordion title="agent_convert / agent_batch_convert / bridge_convert_url не работают">
    Эти инструменты требуют открытого Chrome с установленным расширением Web2MD, а для инструментов `agent_*` — ещё и установленного native host. См. [Agent Bridge](/ru/advanced/agent-bridge) для настройки.
  </Accordion>
  <Accordion title="npx не может разрешить пакет">
    Попробуйте сначала установить пакет глобально: `npm install -g web2md-mcp-server`, затем измените `command` на `web2md-mcp-server` и уберите поле `args`.
  </Accordion>
</AccordionGroup>
