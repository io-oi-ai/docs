## Обзор

Web2MD может отправлять сконвертированный Markdown напрямую в [Obsidian](https://obsidian.md), используя протокол `obsidian://` URI. Это позволяет сохранять веб-контент в хранилище, не покидая браузер.

## Настройка

<Steps>
  <Step title="Включите интеграцию с Obsidian">
    Перейдите в **Settings → Obsidian** и включите переключатель **Enable Obsidian export**.
  </Step>
  <Step title="Укажите название хранилища (опционально)">
    Если у вас несколько хранилищ Obsidian, укажите название нужного, чтобы направить экспорт именно в него. Оставьте поле пустым, чтобы использовать хранилище по умолчанию.
  </Step>
  <Step title="Убедитесь, что Obsidian запущен">
    Для работы протокола URI Obsidian должен быть открыт на вашем компьютере.
  </Step>
</Steps>

## Использование

После конвертации страницы:

1. Нажмите кнопку **Save to Obsidian** на панели результата
2. Obsidian откроется и создаст новую заметку с содержимым в Markdown
3. Заголовок заметки будет установлен по названию страницы

## Как это работает

Web2MD использует [протокол URI](https://help.obsidian.md/Extending+Obsidian/Obsidian+URI) Obsidian:

```
obsidian://new?vault=MyVault&name=Article%20Title&content=...
```

Это создаёт новую заметку в указанном хранилище со сконвертированным Markdown в качестве содержимого.

<Warning>
Протокол `obsidian://` имеет ограничения на длину URL. Очень длинные статьи могут быть обрезаны. Для длинного контента рассмотрите вариант с использованием **Copy** и вставкой напрямую в Obsidian.
</Warning>

## Настройки

| Настройка | Описание |
|---------|-------------|
| **Enable Obsidian export** | Показать/скрыть кнопку «Save to Obsidian» |
| **Vault name** | Целевое хранилище (оставьте пустым для использования хранилища по умолчанию) |
