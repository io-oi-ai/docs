## Обзор

Пользовательские правила извлечения (v0.7) позволяют задавать CSS-селекторы для каждого домена, которые точно определяют, что именно извлекает Web2MD. Если основной контент сайта находится в нестандартном контейнере или есть элементы, которые вы всегда хотите убирать, пользовательские правила дают вам полный контроль.

<Note>
Пользовательские правила — функция, доступная **только на тарифе Pro**.
</Note>

## Создание правила

<Steps>
  <Step title="Откройте настройки">
    Во всплывающем окне Web2MD перейдите в **Settings → Custom Rules**.
  </Step>
  <Step title="Добавьте новое правило">
    Нажмите **New Rule** и укажите домен (например, `example.com`).
  </Step>
  <Step title="Определите селекторы">
    Задайте селектор контента, селекторы для удаления и, при необходимости, селектор заголовка.
  </Step>
  <Step title="Сохраните и конвертируйте">
    Сохраните правило. При следующей конвертации страницы на этом домене ваши пользовательские селекторы будут применены автоматически.
  </Step>
</Steps>

## Поля правила

| Поле | Описание | Пример |
|-------|-------------|---------|
| **Domain** (Домен) | Домен, к которому применяется это правило (точное совпадение) | `docs.example.com` |
| **Content selector** (Селектор контента) | CSS-селектор для основного контейнера контента | `article.post-body` |
| **Remove selectors** (Селекторы для удаления) | CSS-селекторы элементов для удаления (через запятую) | `.ads, .sidebar, .related-posts` |
| **Title selector** (Селектор заголовка) | Переопределяет автоматическое определение заголовка | `h1.entry-title` |

## Пример: очистка блога

Предположим, вы регулярно конвертируете статьи из блога, который оборачивает контент в специфичный div и содержит навязчивые виджеты в боковой панели:

```
Domain:           blog.example.com
Content selector: div.article-content
Remove selectors: .author-bio, .newsletter-signup, .comments-section
Title selector:   h1.post-title
```

С этим правилом Web2MD будет:
1. Извлекать только содержимое внутри `div.article-content`
2. Убирать блок об авторе, форму подписки на рассылку и комментарии
3. Использовать элемент `h1.post-title` в качестве заголовка документа

## Как применяются правила

<CardGroup cols={2}>
  <Card title="Автоматическое сопоставление" icon="bullseye">
    Правила сопоставляются по домену. При конвертации страницы Web2MD проверяет, существует ли пользовательское правило для этого домена, и применяет его до запуска стандартного конвейера извлечения.
  </Card>
  <Card title="Резервное поведение" icon="shield">
    Если ваш селектор контента ничего не находит на странице, Web2MD переключается на стандартную логику извлечения. Ваше правило не сломает конвертацию.
  </Card>
</CardGroup>

## Управление правилами

Все правила управляются в разделе **Settings → Custom Rules** во всплывающем окне расширения.

| Действие | Как выполнить |
|--------|-----|
| **Создать** | Settings → Custom Rules → New Rule |
| **Изменить** | Нажмите на любое существующее правило, чтобы изменить его селекторы |
| **Удалить** | Нажмите на значок корзины рядом с правилом |
| **Проверить** | Перейдите на страницу целевого домена и запустите конвертацию |

<Tip>
Используйте DevTools вашего браузера (`Cmd+Shift+I` или `F12`), чтобы изучить страницу и найти нужные CSS-селекторы перед созданием правила.
</Tip>

## Совместное использование в сообществе

<Info>
Совместное использование правил в сообществе скоро появится. Вы сможете помечать правила как публичные (флаг `isPublic`), чтобы другие пользователи могли пользоваться вашими селекторами для конкретных доменов.
</Info>

## Связь с адаптерами сайтов

Пользовательские правила и [адаптеры сайтов](/ru/advanced/site-adapters) служат разным целям:

| | Адаптеры сайтов | Пользовательские правила |
|-|---------------|-------------|
| **Область применения** | Встроенные, для конкретных платформ (Reddit, GitHub и т. д.) | Любой домен, который вы зададите |
| **Метод** | Использует API платформ и специализированную логику | Только CSS-селекторы |
| **Кто поддерживает** | Команда Web2MD | Вы |
| **Приоритет** | Запускается первым | Запускается, если ни один адаптер не подошёл |

Если для домена существует адаптер сайта, он имеет приоритет над пользовательскими правилами. Пользовательские правила лучше всего подходят для сайтов без выделенного адаптера.
