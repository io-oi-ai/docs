## 개요

Web2MD는 어떤 웹페이지에서도 작동하지만, 일부 사이트는 전용 추출 로직을 통해 더 좋은 결과를 얻을 수 있습니다. Web2MD에는 각 플랫폼의 API나 DOM 구조를 활용해 일반 추출보다 더 나은 Markdown 출력을 만들어내는 **16개의 플랫폼별 추출기**가 포함되어 있습니다.

## 지원 사이트

### API 기반 (탭이 필요 없음)

이 추출기들은 플랫폼의 API를 통해 직접 데이터를 가져오므로, 활성 브라우저 탭이 없어도 작동합니다.

#### Reddit

다음을 포함한 전체 게시물 콘텐츠를 추출합니다.
- 게시물 제목, 작성자, 서브레딧, 점수
- 본문 텍스트 또는 링크 콘텐츠
- 최대 **200개의 댓글**과 4단계까지의 중첩 구조(작성자와 점수 포함)
- 이미지 및 미디어 링크

안정적인 추출을 위해 Reddit의 JSON API(`/.json`)를 사용합니다.

#### Hacker News

Algolia API를 통해 전체 토론 스레드를 추출합니다.
- 게시물 제목, URL, 포인트, 댓글 수
- 작성자와 중첩 구조를 포함한 모든 댓글

#### arXiv

연구 논문을 추출합니다.
- 논문 제목, 저자, 초록
- 연구 논문의 전체 HTML 콘텐츠

#### YouTube

동영상 메타데이터를 추출합니다.
- 동영상 제목과 채널명
- 재생 시간과 조회수
- 영어 자막/캡션 (제공되는 경우)

#### Twitter / X

트윗 콘텐츠를 추출합니다.
- 트윗 텍스트와 미디어
- 작성자 정보와 참여도 지표

### DOM 기반 (브라우저 탭 필요)

이 추출기들은 구조화된 콘텐츠를 위해 활성 탭 내부의 페이지 DOM을 파싱합니다.

#### OpenAI Docs

OpenAI 문서 사이트에서 문서 페이지를 추출합니다.
- 코드 블록과 API 레퍼런스
- 구조화된 내비게이션과 콘텐츠 계층 구조

#### Claude Chat

Claude.ai 채팅 인터페이스에서 대화 내용을 추출합니다.

#### Notion

공개 및 로그인된 Notion 페이지(notion.so, notion.site)에서 콘텐츠를 추출합니다.
- 이중 추출 방식: 블록 단위 DOM 파싱(`data-block-id`)과 컨테이너 폴백
- 제목, 텍스트, 코드 블록, 목록, 토글, 콜아웃, 인용문
- 이미지, 표, 북마크, 임베드

#### Feishu / Lark

Feishu Docx 및 Wiki 페이지(feishu.cn, larksuite.com)에서 콘텐츠를 추출합니다.
- 이중 추출 방식: PageMain 블록 모델과 DOM 폴백
- 제목, 텍스트, 코드 블록, 목록, 표, 이미지
- 콜아웃과 체크박스

#### Medium

Medium의 DOM에서 기사 콘텐츠를 추출합니다.
- 작성자, 게시일, 읽는 시간
- 전체 기사 콘텐츠

#### Substack

뉴스레터 게시물을 추출합니다.
- 작성자, 날짜, 전체 기사 콘텐츠

#### Wikipedia

기사 콘텐츠를 추출합니다.
- 섹션, 표, 참고 문헌
- 내비게이션과 사이드바의 잡음을 제거

#### DEV.to

개발자 블로그 게시물을 추출합니다.
- 태그, 반응(reactions), 댓글

#### Product Hunt

제품 페이지를 추출합니다.
- 제품 설명과 제작자 정보
- 토론 스레드

#### Stack Overflow

Q&A 콘텐츠를 추출합니다.
- 질문 제목, 본문, 태그, 투표 수
- 채택된 답변 (강조 표시)
- 투표 수 기준 상위 답변

#### GitHub

다음을 포함한 Issues 및 Pull Requests를 추출합니다.
- 제목, 상태(open/closed/merged), 라벨
- 원본 설명/본문
- 작성자 정보를 포함한 최대 **30개의 댓글**

구조화된 데이터를 위해 GitHub의 REST API를 사용합니다.

## 어댑터 작동 방식

페이지를 변환할 때 Web2MD는 자동으로 다음을 수행합니다.

1. URL로부터 **사이트를 감지**합니다
2. 존재하는 경우 **최적의 어댑터를 선택**합니다
3. 사이트별 방식으로 **콘텐츠를 추출**합니다
4. 어댑터가 실패하면 일반 추출로 **폴백**합니다

별도로 설정할 필요는 없습니다 — 어댑터는 자동으로 적용됩니다.

<Info>
Site adapters는 모든 사용자(Free 및 Pro)가 이용할 수 있습니다. 메인 변환 파이프라인보다 먼저 실행되므로 플랜과 관계없이 작동합니다.
</Info>

## 비교

| 기능 | 일반 추출 | Site adapter |
|---------|-------------------|--------------|
| 콘텐츠 정확도 | 기사에는 적합 | 사이트 구조에 최적화 |
| 댓글/답글 | 추출되지 않음 | 포함됨 (Reddit, GitHub, SO) |
| 메타데이터 | 기본 (제목, URL) | 풍부함 (작성자, 점수, 상태) |
| 미디어 | 이미지만 | 동영상, 자막, 임베드 |
| 신뢰성 | 페이지 구조에 따라 다름 | 안정적인 API 사용 |
