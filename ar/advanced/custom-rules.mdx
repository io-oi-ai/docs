## نظرة عامة

تتيح لك قواعد الاستخراج المخصصة (Custom Extraction Rules) (v0.7) تعريف محدِّدات CSS لكل نطاق تتحكم بدقة فيما يستخرجه Web2MD. إذا كان المحتوى الرئيسي لموقع ما يقع داخل حاوية غير معتادة، أو إذا كانت هناك عناصر تريد دائمًا إزالتها، تمنحك القواعد المخصصة تحكمًا كاملًا.

<Note>
القواعد المخصصة ميزة **حصرية لخطة Pro**.
</Note>

## إنشاء قاعدة

<Steps>
  <Step title="افتح الإعدادات">
    في نافذة Web2MD المنبثقة، انتقل إلى **Settings → Custom Rules**.
  </Step>
  <Step title="أضف قاعدة جديدة">
    اضغط **New Rule** وأدخل النطاق (مثل `example.com`).
  </Step>
  <Step title="عرّف محدِّداتك">
    اضبط محدِّد المحتوى، ومحدِّدات الإزالة، واختياريًا محدِّد العنوان.
  </Step>
  <Step title="احفظ وحوّل">
    احفظ القاعدة. في المرة القادمة التي تحوّل فيها صفحة على ذلك النطاق، ستُطبَّق محدِّداتك المخصصة تلقائيًا.
  </Step>
</Steps>

## حقول القاعدة

| الحقل | الوصف | مثال |
|-------|-------------|---------|
| **النطاق (Domain)** | النطاق الذي تنطبق عليه هذه القاعدة (تطابق دقيق) | `docs.example.com` |
| **محدِّد المحتوى** | محدِّد CSS لحاوية المحتوى الرئيسي | `article.post-body` |
| **محدِّدات الإزالة** | محدِّدات CSS للعناصر المراد إزالتها (مفصولة بفواصل) | `.ads, .sidebar, .related-posts` |
| **محدِّد العنوان** | تجاوز الكشف التلقائي للعنوان | `h1.entry-title` |

## مثال: تنظيف مدونة

لنفترض أنك تحوّل بانتظام مقالات من مدونة تضع المحتوى داخل عنصر div محدد وتتضمن أدوات جانبية مزعجة:

```
Domain:           blog.example.com
Content selector: div.article-content
Remove selectors: .author-bio, .newsletter-signup, .comments-section
Title selector:   h1.post-title
```

مع هذه القاعدة، سيقوم Web2MD بما يلي:
1. استخراج المحتوى فقط داخل `div.article-content`
2. إزالة السيرة الذاتية للكاتب، ونموذج الاشتراك في النشرة البريدية، وقسم التعليقات
3. استخدام عنصر `h1.post-title` كعنوان للمستند

## كيف تُطبَّق القواعد

<CardGroup cols={2}>
  <Card title="مطابقة تلقائية" icon="bullseye">
    تُطابَق القواعد حسب النطاق. عند تحويل صفحة، يتحقق Web2MD مما إذا كانت هناك قاعدة مخصصة لذلك النطاق ويطبّقها قبل خط أنابيب الاستخراج القياسي.
  </Card>
  <Card title="السلوك الاحتياطي" icon="shield">
    إذا لم يطابق محدِّد المحتوى الخاص بك أي شيء في الصفحة، يعود Web2MD إلى منطق الاستخراج الافتراضي. لن تُفسِد قاعدتك عمليات التحويل.
  </Card>
</CardGroup>

## إدارة القواعد

تُدار جميع القواعد في **Settings → Custom Rules** داخل نافذة الإضافة المنبثقة.

| الإجراء | الطريقة |
|--------|-----|
| **إنشاء** | Settings → Custom Rules → New Rule |
| **تعديل** | اضغط على أي قاعدة موجودة لتعديل محدِّداتها |
| **حذف** | اضغط أيقونة سلة المهملات بجانب القاعدة |
| **اختبار** | انتقل إلى صفحة على النطاق المستهدف وشغّل عملية تحويل |

<Tip>
استخدم أدوات المطوّر في متصفحك (`Cmd+Shift+I` أو `F12`) لفحص الصفحة وإيجاد محدِّدات CSS المناسبة قبل إنشاء قاعدة.
</Tip>

## المشاركة المجتمعية

<Info>
مشاركة القواعد المجتمعية قادمة قريبًا. ستتمكن من تمييز القواعد كعامة (علامة `isPublic`) حتى يستفيد مستخدمون آخرون من محدِّداتك الخاصة بالنطاق.
</Info>

## العلاقة بمحوّلات المواقع

تخدم القواعد المخصصة و[محوّلات المواقع](/ar/advanced/site-adapters) أغراضًا مختلفة:

| | محوّلات المواقع | القواعد المخصصة |
|-|---------------|-------------|
| **النطاق** | مدمجة، لمنصات محددة (Reddit، GitHub، إلخ) | أي نطاق تعرّفه |
| **الطريقة** | تستخدم واجهات برمجة تطبيقات المنصات ومنطقًا متخصصًا | محدِّدات CSS فقط |
| **الصيانة من قِبل** | فريق Web2MD | أنت |
| **الأولوية** | تعمل أولًا | تعمل إذا لم يطابق أي محوّل |

إذا وُجد محوّل موقع لنطاق ما، فإنه يأخذ الأولوية على القواعد المخصصة. القواعد المخصصة هي الأنسب للمواقع التي لا يوجد لها محوّل مخصص.
