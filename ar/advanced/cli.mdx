## نظرة عامة

يتيح لك CLI الخاص بـ Web2MD تحويل أي رابط إلى Markdown نظيف مباشرة من الطرفية (Terminal). مرّر المخرجات إلى نماذج LLM، أو عالج قوائم روابط دفعة واحدة، أو أدرج المحتوى في مخزن Obsidian الخاص بك — كل ذلك دون فتح متصفح.

```bash
npx web2md https://example.com/article
```

<Note>
  يتطلب CLI إصدار **Node.js 18** أو أحدث. شغّل `node -v` للتحقق من إصدارك.
</Note>

## التثبيت

لا حاجة لأي تثبيت — فقط شغّله باستخدام `npx`:

```bash
npx web2md <url> [options]
```

أو ثبّته عالميًا لبدء تشغيل أسرع:

```bash
npm install -g web2md-cli
web2md <url> [options]
```

## أوضاع التشغيل

يعمل CLI الخاص بـ Web2MD بثلاثة أوضاع حسب إعداداتك:

<CardGroup cols={3}>
  <Card title="محلي" icon="laptop">
    **الوضع الافتراضي.** لا يتطلب مفتاح API. يجلب الصفحات ويحوّلها محليًا. يعمل مع معظم المواقع العامة.
  </Card>
  <Card title="خادم (Server)" icon="server">
    **مع مفتاح API.** اضبط `WEB2MD_API_KEY` لفتح دعم Reddit وFandom/Wikia وغيرها من المواقع المقيَّدة التي تتطلب معالجة من جهة الخادم.
  </Card>
  <Card title="جسر (Bridge)" icon="bridge">
    **مع علامة `--bridge`.** يستخدم إضافة Chrome لجلب الصفحات التي تُعرَض عبر JS أو المحمية بتسجيل دخول والتي لا يمكن للجلب الثابت الوصول إليها.
  </Card>
</CardGroup>

## العلامات (Flags)

| العلامة | الوصف |
|------|-------------|
| `--no-images` | إزالة مراجع الصور من المخرجات |
| `--no-links` | إزالة الروابط التشعبية من المخرجات |
| `--meta` | إضافة ترويسة YAML (العنوان، المصدر، عدد الكلمات، عدد الرموز، وقت القراءة، التاريخ) |
| `--json` | إخراج بصيغة JSON `{ markdown, metadata }` |
| `-o, --output <file>` | كتابة المخرجات إلى ملف |
| `--output-dir <dir>` | كتابة كل رابط إلى ملف `.md` منفصل |
| `--batch <file>` | قراءة الروابط من ملف (رابط واحد في كل سطر، `#` للتعليقات) |
| `--vault <dir>` | وضع مخزن Obsidian: يحفظ في `<dir>/raw/` ويحدّث `<dir>/INDEX.md` |
| `--concurrency <n>` | الحد الأقصى لعمليات الجلب المتوازية (الافتراضي: 3، الحد الأقصى: 20) |
| `--bridge` | استخدام إضافة Chrome للمواقع المعروضة عبر JS أو المحمية بتسجيل دخول |
| `-q, --quiet` | إخفاء رسائل التقدّم |

## الأوامر

إلى جانب تحويل الروابط، يحتوي CLI على أمرين فرعيين للمصادقة ومزامنة المكتبة:

| الأمر | الوصف |
|---------|-------------|
| `web2md login` | المصادقة مرة واحدة وحفظ مفتاح API الخاص بك في `~/.web2md/config.json` |
| `web2md sync [dir]` | مزامنة مكتبة Web2MD الخاصة بك إلى مجلد محلي كملفات Markdown |

### login

```bash
npx web2md-cli login
```

يبدأ تدفق تفويض عبر الجهاز (device authorization) قائم على المتصفح: يفتح CLI عنوان `https://web2md.org` ويعرض رمزًا للموافقة عليه. بمجرد الموافقة، يُحفظ مفتاح API الخاص بك في `~/.web2md/config.json` — دون الحاجة لتصدير `WEB2MD_API_KEY` في كل صدفة (shell). يقرأ [خادم MCP](/ar/advanced/mcp-server#authentication) من نفس الملف، لذا فإن تسجيل الدخول هنا مرة واحدة يُصادق أيضًا استدعاءات أدوات MCP.

هل تفضّل لصق مفتاح مباشرة بدلاً من تدفق المتصفح؟

```bash
npx web2md-cli login --key
```

يطلب هذا مفتاحًا (`w2m_...`) عبر الإدخال القياسي (stdin) ويحفظه بنفس الطريقة. احصل على مفتاح من [لوحة التحكم](https://web2md.org/dashboard/api-keys).

### sync

```bash
npx web2md-cli sync [dir] [--tag <t>] [--full]
```

يزامن مكتبتك — كل صفحة قمت بحفظها عبر الإضافة أو لوحة التحكم أو API — إلى مجلد محلي كملفات Markdown مع بيانات وصفية أمامية (frontmatter)، منظّمة حسب الوسم. يتطلب تسجيل الدخول (أو ضبط `WEB2MD_API_KEY`).

| العلامة | الوصف |
|------|-------------|
| `dir` | المجلد الهدف (الافتراضي: `./web2md-vault`) |
| `--tag <t>` | مزامنة العناصر ذات هذا الوسم فقط |
| `--full` | تجاهل المؤشر التزايدي وإعادة مسح السجل الكامل |

المزامنة تزايدية افتراضيًا: تتبع مؤشرًا في `<dir>/.web2md-sync.json` وتسحب فقط العناصر الجديدة في عمليات التشغيل اللاحقة. وجّه Obsidian (أو أي أداة تقرأ Markdown) نحو المجلد الهدف لتصفح مكتبتك كملاحظات.

```bash
npx web2md-cli login
npx web2md-cli sync ~/Documents/Web2MD-Library
```

## متغيرات البيئة

| المتغير | الوصف |
|----------|-------------|
| `WEB2MD_API_KEY` | مفتاح API (`w2m_xxx`) لـ Reddit والمواقع المقيَّدة و`sync` — له الأولوية على `~/.web2md/config.json` |
| `WEB2MD_API_URL` | تجاوز عنوان API الأساسي |
| `WEB2MD_EXTENSION_ID` | تجاوز معرّف إضافة Chrome لوضع `--bridge` |

<Tip>
  أضف هذه المتغيرات إلى ملف تهيئة الصدفة الخاص بك (`~/.zshrc` أو `~/.bashrc`) لتبقى ثابتة بين الجلسات:
  ```bash
  export WEB2MD_API_KEY="w2m_your_key_here"
  ```
</Tip>

## أمثلة على الاستخدام

### تحويل أساسي

```bash
npx web2md https://example.com/article
```

يطبع Markdown إلى الإخراج القياسي (stdout).

### التمرير إلى نموذج LLM

```bash
npx web2md https://react.dev/learn/thinking-in-react | llm "Summarize this page"
```

```bash
npx web2md https://docs.python.org/3/tutorial/classes.html | claude "Explain the key concepts"
```

### الحفظ في ملف

```bash
npx web2md https://example.com/article -o article.md
```

```bash
npx web2md https://example.com/article --meta -o article.md
```

تضيف علامة `--meta` ترويسة YAML تتضمن العنوان، رابط المصدر، عدد الكلمات، عدد الرموز، وقت القراءة، والتاريخ.

### المعالجة الدفعية من ملف

أنشئ ملفًا باسم `urls.txt`:

```text
# Research papers
https://arxiv.org/abs/2301.00001
https://arxiv.org/abs/2301.00002

# Blog posts
https://example.com/blog/post-1
https://example.com/blog/post-2
```

ثم شغّل:

```bash
npx web2md --batch urls.txt --output-dir ./research --concurrency 5
```

يُحفظ كل رابط كملف `.md` منفصل داخل مجلد `./research`.

### الإدراج في مخزن Obsidian

```bash
npx web2md --batch urls.txt --vault ~/Documents/MyVault
```

يحفظ هذا كل صفحة في `~/Documents/MyVault/raw/` ويحدّث `~/Documents/MyVault/INDEX.md` بروابط لجميع الصفحات المحوَّلة.

### Reddit بمفتاح API

```bash
export WEB2MD_API_KEY="w2m_your_key_here"
npx web2md https://www.reddit.com/r/LocalLLaMA/comments/example
```

<Warning>
  يتطلب Reddit مفتاح API صالحًا. بدونه، ستفشل روابط Reddit بسبب قيود Reddit المضادة للبوتات.
</Warning>

### وضع Bridge

استخدم إضافة Chrome للتعامل مع الصفحات المعروضة عبر JS أو المحمية بتسجيل دخول:

```bash
npx web2md --bridge https://app.example.com/dashboard
```

<Note>
  يتطلب وضع Bridge تثبيت إضافة Web2MD لمتصفح Chrome وتشغيل Chrome. يتواصل CLI مع الإضافة عبر بروتوكول المراسلة الأصلي (native messaging) الخاص بـ Chrome.
</Note>

### إخراج JSON

```bash
npx web2md --json https://example.com/article
```

يُرجع مخرجات منظَّمة:

```json
{
  "markdown": "# Article Title\n\nContent here...",
  "metadata": {
    "title": "Article Title",
    "source": "https://example.com/article",
    "wordCount": 1250,
    "tokenCount": 1680,
    "readingTime": 5,
    "date": "2026-04-11T10:30:00.000Z"
  }
}
```

مفيدة للاستهلاك البرمجي أو للتمرير إلى `jq`:

```bash
npx web2md --json https://example.com/article | jq '.metadata.tokenCount'
```
