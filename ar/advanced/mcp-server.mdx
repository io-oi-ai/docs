## نظرة عامة

يُتيح خادم Web2MD MCP استخدام Web2MD كمجموعة أدوات يمكن لمساعدات AI استدعاءها مباشرة، باستخدام [Model Context Protocol (MCP)](https://modelcontextprotocol.io) — وهو معيار مفتوح لربط نماذج AI بالأدوات ومصادر البيانات الخارجية.

بعد الإعداد، يمكنك أن تطلب من Claude أو Cursor أو Windsurf تحويل الصفحات، أو البحث في مكتبتك ومزامنتها، أو التحويل الجماعي عبر إضافة Chrome، أو سحب Skill محفوظة — دون مغادرة المحادثة.

<Note>
  يتطلب خادم MCP **مفتاح API لـ Web2MD** (خطة PRO). أنشئ واحدًا من [لوحة التحكم](https://web2md.org/dashboard/api-keys)، أو نفّذ `npx web2md-cli login` للمصادقة مرة واحدة ومشاركة المفتاح مع CLI تلقائيًا (راجع [المصادقة](#authentication) أدناه).
</Note>

## الأدوات المتاحة

يوفر خادم MCP 11 أداة، مصنّفة حسب وظيفتها:

| الأداة | الوصف |
|------|-------------|
| `convert_url` | تحويل أي رابط إلى Markdown نظيف عبر المحوّل على جانب الخادم |
| `bridge_convert_url` | تحويل رابط عبر إضافة Chrome الخاصة بك عبر HTTP — لـ Reddit، أو الصفحات المُصيَّرة بـ JS، أو الصفحات المحمية بتسجيل الدخول |
| `agent_convert` | تحويل رابط عبر إضافة Chrome باستخدام Native Messaging (نفس حالات استخدام `bridge_convert_url` لكن بنقل مختلف) |
| `agent_batch_convert` | تحويل جماعي لما يصل إلى 50 رابطًا عبر إضافة Chrome باستخدام Native Messaging |
| `search_library` | البحث في مكتبتك المحفوظة حسب الكلمة المفتاحية و/أو الوسم |
| `get_library_items` | جلب Markdown الكامل لعناصر المكتبة عبر المُعرِّف (id) |
| `sync_library` | مزامنة مكتبتك إلى مجلد محلي كملفات Markdown |
| `list_skills` | سرد الـ Skills المحفوظة في حساب Web2MD السحابي الخاص بك |
| `get_skill` | جلب المحتوى الكامل لـ Skill واحدة عبر المُعرِّف أو الاسم |
| `semantic_search` | **مهملة (Deprecated)** — استخدم `search_library` و`get_library_items` بدلاً منها |
| `get_conversion` | **مهملة (Deprecated)** — استخدم `search_library` و`get_library_items` بدلاً منها |

### convert_url

يجلب رابطًا على جانب الخادم ويُرجع محتوى الصفحة كـ Markdown، مع العنوان وعدد الكلمات ووقت القراءة. يدعم نفس خيارات التحويل المتوفرة في REST API (`includeImages`, `includeLinks`).

**مثال على طلب:** _"حوّل هذا الرابط إلى markdown: https://docs.example.com/getting-started"_

### bridge_convert_url

يحوّل رابطًا عن طريق جلب HTML الخاص به عبر إضافة Chrome العاملة لديك (عبر جسر HTTP محلي)، ثم يحوّل ذلك الـ HTML إلى Markdown على جانب الخادم. استخدم هذا للصفحات التي لا يمكن للخادم الوصول إليها مباشرة — Reddit، أو المواقع المحمية بتسجيل الدخول، أو الصفحات المُصيَّرة بكثافة عبر JS. يتطلب فتح Chrome مع تثبيت إضافة Web2MD؛ يتم اكتشاف معرّف الإضافة تلقائيًا من مفتاح API الخاص بك.

**مثال على طلب:** _"استخدم الجسر لتحويل موضوع Reddit هذا: https://reddit.com/r/..."_

### agent_convert

يحوّل رابطًا بنفس طريقة `bridge_convert_url` — عبر إضافة Chrome الخاصة بك — لكن عبر قناة [Agent Bridge](/ar/advanced/agent-bridge) بواسطة Native Messaging بدلاً من HTTP. يقبل معامل `skill` اختياريًا: مرّر معرّف أو اسم Skill ليتم إلحاق محتواها بـ Markdown المُرجَع حتى يتمكن الوكيل من التصرف بناءً عليها فورًا. يتطلب تشغيل Chrome مع تثبيت الإضافة والمضيف الأصلي (native host).

**مثال على طلب:** _"حوّل هذه الصفحة مع تطبيق مهارة التلخيص: https://..."_

### agent_batch_convert

نفس نقل `agent_convert`، لكنه يأخذ مصفوفة تصل إلى 50 رابطًا ويحوّلها بالتتابع في استدعاء واحد. الأفضل لسحب العديد من مواضيع Reddit أو الصفحات المُصيَّرة بـ JS دفعة واحدة.

**مثال على طلب:** _"حوّل هذه الروابط الـ 12 إلى Markdown دفعة واحدة: [...]"_

### search_library

يبحث في مكتبة Web2MD الخاصة بك — كل صفحة قمت بحفظها — حسب الكلمة المفتاحية (يطابق العنوان والمحتوى الكامل للصفحة) و/أو الوسم، من الأحدث إلى الأقدم. يُرجع بيانات وصفية مختصرة (id، العنوان، الرابط، التاريخ، الوسوم)، وليس Markdown الكامل. حسابات الخطة المجانية ترى فقط أحدث العناصر في النتائج؛ تخبرك الأداة عندما يتم اقتطاع النتائج.

**مثال على طلب:** _"ابحث في مكتبتي عن أي شيء متعلق بالمصادقة"_

### get_library_items

يجلب محتوى Markdown الكامل لعناصر المكتبة عبر المُعرِّف (حتى 20 عنصرًا لكل استدعاء — استخدم `search_library` أولاً للعثور على المعرّفات). قد تكون الاستجابات كبيرة؛ للصفحات الطويلة، اجمع 5 معرّفات أو أقل لكل استدعاء.

**مثال على طلب:** _"احصل على المحتوى الكامل لعنصري المكتبة abc123 وdef456"_

### sync_library

يزامن مكتبتك بأكملها إلى مجلد محلي كملفات Markdown، منظّمة حسب الوسم مع بيانات وصفية أمامية (frontmatter). تزايدية افتراضيًا (تتبع مؤشرًا في `<dir>/.web2md-sync.json`)؛ مرّر `full: true` لإعادة مسح كل شيء. هذه هي نفس المزامنة التي ينفذها `web2md sync` من [CLI](/ar/advanced/cli) — استخدم أي واجهة تناسب سير عملك.

**مثال على طلب:** _"زامن مكتبتي إلى ~/notes/web2md، بوسم 'research' فقط"_

### list_skills

يسرد الـ Skills المحفوظة في حساب Web2MD السحابي الخاص بك كمصفوفة مختصرة (id، الاسم، الوصف). استخدم `get_skill` لجلب واحدة كاملة.

**مثال على طلب:** _"ما هي المهارات المحفوظة لدي؟"_

### get_skill

يجلب المحتوى الكامل بأسلوب SKILL.md لـ Skill واحدة عبر المُعرِّف أو الاسم — تعليماتها ووصفها ومحتواها بالكامل. استخدم `list_skills` أولاً إذا كنت لا تعرف المعرّف.

**مثال على طلب:** _"احصل على مهارة 'summarize' الخاصة بي"_

### semantic_search / get_conversion (مهملة)

هاتان الأداتان تسبقان المكتبة. لا تزالان تعملان لكنهما تريان فقط التحويلات المحفوظة عبر واجهة السجل القديمة، وليس المكتبة الكاملة — يجب أن تستخدم عمليات الدمج الجديدة `search_library` و`get_library_items` بدلاً من ذلك.

## المصادقة

يحسم الخادم مفتاح API بهذا الترتيب:

1. متغير البيئة `WEB2MD_API_KEY`، إن وُجد.
2. وإلا، `~/.web2md/config.json` — نفس الملف الذي يكتبه `npx web2md-cli login` (تدفق تسجيل الدخول عبر الجهاز في [CLI](/ar/advanced/cli)). نفّذ `login` مرة واحدة وسيلتقط كل من CLI وخادم MCP المفتاح تلقائيًا، دون الحاجة لكتلة `env`.

إذا لم يتم العثور على أي منهما، يبدأ الخادم لكن كل استدعاء أداة يُرجع خطأ مصادقة مع تلميح لضبط المفتاح أو تنفيذ `login`.

## الإعداد

<Tabs>
  <Tab title="Claude Desktop">
    افتح ملف إعدادات Claude Desktop:

    - **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
    - **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

    أضف إدخال `web2md` ضمن `mcpServers`:

    ```json
    {
      "mcpServers": {
        "web2md": {
          "command": "npx",
          "args": ["-y", "web2md-mcp-server"],
          "env": {
            "WEB2MD_API_KEY": "w2m_your_key_here"
          }
        }
      }
    }
    ```

    إذا سبق لك تنفيذ `npx web2md-cli login`، يمكنك حذف كتلة `env` بالكامل — يقرأ الخادم المفتاح من `~/.web2md/config.json`.

    أعد تشغيل Claude Desktop. يجب أن تظهر أدوات Web2MD في منتقي الأدوات.
  </Tab>
  <Tab title="Cursor">
    افتح ملف إعدادات Cursor MCP في `~/.cursor/mcp.json` وأضف:

    ```json
    {
      "mcpServers": {
        "web2md": {
          "command": "npx",
          "args": ["-y", "web2md-mcp-server"],
          "env": {
            "WEB2MD_API_KEY": "w2m_your_key_here"
          }
        }
      }
    }
    ```

    أعد تشغيل Cursor لتحميل الخادم الجديد.
  </Tab>
  <Tab title="Windsurf">
    افتح ملف إعدادات Windsurf MCP في `~/.windsurf/mcp.json` وأضف:

    ```json
    {
      "mcpServers": {
        "web2md": {
          "command": "npx",
          "args": ["-y", "web2md-mcp-server"],
          "env": {
            "WEB2MD_API_KEY": "w2m_your_key_here"
          }
        }
      }
    }
    ```

    أعد تشغيل Windsurf لتحميل الخادم الجديد.
  </Tab>
</Tabs>

## أمثلة الاستخدام

بمجرد تشغيل خادم MCP، يمكنك استخدام اللغة الطبيعية في مساعد AI الخاص بك:

<AccordionGroup>
  <Accordion title="تحويل صفحة إلى Markdown">
    **أنت:** حوّل هذا الرابط إلى markdown: https://react.dev/learn/thinking-in-react

    **المساعد:** _(يستدعي `convert_url`)_ إليك Markdown المحوَّل لصفحة "Thinking in React"...
  </Accordion>
  <Accordion title="البحث والسحب من مكتبتك">
    **أنت:** ابحث في مكتبتي عن أي شيء متعلق بالمصادقة، ثم احصل على المحتوى الكامل للأحدث

    **المساعد:** _(يستدعي `search_library` ثم `get_library_items`)_ وجدت 3 عناصر متعلقة بالمصادقة. إليك المحتوى الكامل للأحدث، "OAuth 2.0 Guide"...
  </Accordion>
  <Accordion title="مزامنة مكتبتك محليًا">
    **أنت:** زامن مكتبتي إلى ~/notes/web2md

    **المساعد:** _(يستدعي `sync_library`)_ تمت مزامنة 42 عنصرًا جديدًا، و118 محدّثًا مسبقًا، و0 فشل.
  </Accordion>
  <Accordion title="التحويل الجماعي عبر الإضافة">
    **أنت:** حوّل هذه المواضيع الـ 8 من Reddit دفعة واحدة: [...]

    **المساعد:** _(يستدعي `agent_batch_convert`)_ تم تحويل 8/8 روابط بنجاح.
  </Accordion>
  <Accordion title="تطبيق Skill محفوظة">
    **أنت:** ما هي المهارات المتوفرة لدي، وهل يمكنك تطبيق "summarize" على هذه الصفحة: https://...

    **المساعد:** _(يستدعي `list_skills` ثم `agent_convert` مع `skill: "summarize"`)_ إليك الصفحة محوَّلة إلى Markdown، مع تطبيق مهارة التلخيص أسفلها...
  </Accordion>
</AccordionGroup>

<Tip>
  اجمع بين الأدوات في محادثة واحدة. على سبيل المثال: _"ابحث في مكتبتي عن وثائق React، ثم احصل على المحتوى الكامل للأحدث."_
</Tip>

## استكشاف الأخطاء وإصلاحها

<AccordionGroup>
  <Accordion title="الأدوات لا تظهر بعد الإعداد">
    تأكد من أنك أعدت تشغيل التطبيق بعد تعديل ملف الإعدادات. تحقق من أن `npx -y web2md-mcp-server` يعمل بنجاح في الطرفية لديك.
  </Accordion>
  <Accordion title="أخطاء المصادقة">
    تحقق من أن `WEB2MD_API_KEY` مفتاح `w2m_` صالح، أو أن `~/.web2md/config.json` يحتوي على واحد (أعد تنفيذ `npx web2md-cli login` إن لم تكن متأكدًا). يمكنك التحقق من مفتاحك على [صفحة مفاتيح API](https://web2md.org/dashboard/api-keys).
  </Accordion>
  <Accordion title="فشل agent_convert / agent_batch_convert / bridge_convert_url">
    تتطلب هذه الأدوات فتح Chrome مع تثبيت إضافة Web2MD، وبالنسبة لأدوات `agent_*` — تثبيت المضيف الأصلي (native host) أيضًا. راجع [Agent Bridge](/ar/advanced/agent-bridge) للإعداد.
  </Accordion>
  <Accordion title="فشل npx في حل الحزمة">
    جرّب تثبيت الحزمة عالميًا أولاً: `npm install -g web2md-mcp-server`، ثم غيّر `command` إلى `web2md-mcp-server` واحذف حقل `args`.
  </Accordion>
</AccordionGroup>
