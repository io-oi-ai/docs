## 概要

Web2MD CLI を使えば、ターミナルから直接、任意の URL をクリーンな Markdown に変換できます。出力を LLM にパイプしたり、URL リストをバッチ処理したり、ブラウザを開かずに Obsidian vault へコンテンツを取り込んだりできます。

```bash
npx web2md https://example.com/article
```

<Note>
  CLI には **Node.js 18 以上**が必要です。`node -v` でバージョンを確認してください。
</Note>

## インストール

インストールは不要です — `npx` でそのまま実行できます。

```bash
npx web2md <url> [options]
```

または、起動を高速化するためにグローバルインストールします。

```bash
npm install -g web2md-cli
web2md <url> [options]
```

## モード

Web2MD CLI は、設定に応じて 3 つのモードで動作します。

<CardGroup cols={3}>
  <Card title="Local" icon="laptop">
    **デフォルトモード。** API キーは不要です。ページをローカルで取得・変換します。ほとんどの公開サイトで動作します。
  </Card>
  <Card title="Server" icon="server">
    **API キー使用時。** `WEB2MD_API_KEY` を設定すると、Reddit、Fandom/Wikia など、サーバー側の処理が必要な制限付きサイトを利用できます。
  </Card>
  <Card title="Bridge" icon="bridge">
    **`--bridge` フラグ使用時。** 静的な取得ではアクセスできない、JS レンダリングされたページやログインが必要なページを、Chrome 拡張機能を使って取得します。
  </Card>
</CardGroup>

## フラグ

| フラグ | 説明 |
|------|-------------|
| `--no-images` | 出力から画像参照を除去 |
| `--no-links` | 出力からハイパーリンクを除去 |
| `--meta` | YAML フロントマターを追加(title、source、wordCount、tokenCount、readingTime、date) |
| `--json` | JSON 形式で出力 `{ markdown, metadata }` |
| `-o, --output <file>` | 出力をファイルに書き込み |
| `--output-dir <dir>` | 各 URL を個別の `.md` ファイルに書き込み |
| `--batch <file>` | ファイルから URL を読み込み(1 行に 1 つ、`#` はコメント) |
| `--vault <dir>` | Obsidian vault モード: `<dir>/raw/` に保存し、`<dir>/INDEX.md` を更新 |
| `--concurrency <n>` | 並列取得の最大数(デフォルト: 3、最大: 20) |
| `--bridge` | JS レンダリングやログインが必要なサイトに Chrome 拡張機能を使用 |
| `-q, --quiet` | 進捗メッセージを抑制 |

## コマンド

URL の変換以外に、CLI には認証と Library 同期のための 2 つのサブコマンドがあります。

| コマンド | 説明 |
|---------|-------------|
| `web2md login` | 一度認証し、API キーを `~/.web2md/config.json` に保存 |
| `web2md sync [dir]` | Web2MD Library をローカルディレクトリに Markdown ファイルとして同期 |

### login

```bash
npx web2md-cli login
```

ブラウザベースのデバイス認可フローを開始します。CLI が `https://web2md.org` を開き、承認用のコードを表示します。承認されると、API キーが `~/.web2md/config.json` に保存されるため、シェルごとに `WEB2MD_API_KEY` をエクスポートする必要がなくなります。[MCP サーバー](/ja/advanced/mcp-server#認証)も同じファイルを読み込むため、ここで一度ログインすれば MCP ツール呼び出しも認証されます。

ブラウザフローの代わりにキーを直接貼り付けたい場合は、以下を使用してください。

```bash
npx web2md-cli login --key
```

これにより標準入力でキー(`w2m_...`)の入力が求められ、同じ方法で保存されます。キーは[ダッシュボード](https://web2md.org/dashboard/api-keys)から取得できます。

### sync

```bash
npx web2md-cli sync [dir] [--tag <t>] [--full]
```

拡張機能・ダッシュボード・API 経由で保存したすべてのページからなる Library を、フロントマター付きの Markdown ファイルとしてタグ別にローカルディレクトリへ同期します。ログイン済み(または `WEB2MD_API_KEY` 設定済み)であることが必要です。

| フラグ | 説明 |
|------|-------------|
| `dir` | 同期先ディレクトリ(デフォルト: `./web2md-vault`) |
| `--tag <t>` | このタグが付いたアイテムのみ同期 |
| `--full` | 差分同期用のカーソルを無視し、履歴全体を再スキャン |

同期はデフォルトで差分方式です。`<dir>/.web2md-sync.json` にカーソルを記録し、次回以降の実行では新規アイテムのみを取得します。Obsidian(または Markdown を読めるツール)の対象にこのディレクトリを指定すれば、Library をノートとして閲覧できます。

```bash
npx web2md-cli login
npx web2md-cli sync ~/Documents/Web2MD-Library
```

## 環境変数

| 変数 | 説明 |
|----------|-------------|
| `WEB2MD_API_KEY` | Reddit、制限付きサイト、`sync` 用の API キー(`w2m_xxx`) — `~/.web2md/config.json` より優先されます |
| `WEB2MD_API_URL` | API のベース URL を上書き |
| `WEB2MD_EXTENSION_ID` | `--bridge` モード用の Chrome 拡張機能 ID を上書き |

<Tip>
  セッションをまたいで永続化するには、シェルプロファイル(`~/.zshrc` や `~/.bashrc`)に以下を追加してください。
  ```bash
  export WEB2MD_API_KEY="w2m_your_key_here"
  ```
</Tip>

## 使用例

### 基本的な変換

```bash
npx web2md https://example.com/article
```

Markdown を標準出力に表示します。

### LLM へのパイプ

```bash
npx web2md https://react.dev/learn/thinking-in-react | llm "Summarize this page"
```

```bash
npx web2md https://docs.python.org/3/tutorial/classes.html | claude "Explain the key concepts"
```

### ファイルへの保存

```bash
npx web2md https://example.com/article -o article.md
```

```bash
npx web2md https://example.com/article --meta -o article.md
```

`--meta` フラグは、タイトル・ソース URL・文字数・トークン数・読了時間・日付を含む YAML フロントマターを先頭に追加します。

### ファイルからのバッチ処理

`urls.txt` というファイルを作成します。

```text
# Research papers
https://arxiv.org/abs/2301.00001
https://arxiv.org/abs/2301.00002

# Blog posts
https://example.com/blog/post-1
https://example.com/blog/post-2
```

そして実行します。

```bash
npx web2md --batch urls.txt --output-dir ./research --concurrency 5
```

各 URL は `./research` ディレクトリ内に個別の `.md` ファイルとして保存されます。

### Obsidian vault への取り込み

```bash
npx web2md --batch urls.txt --vault ~/Documents/MyVault
```

各ページが `~/Documents/MyVault/raw/` に保存され、`~/Documents/MyVault/INDEX.md` が変換済みページすべてへのリンクで更新されます。

### API キーを使った Reddit の変換

```bash
export WEB2MD_API_KEY="w2m_your_key_here"
npx web2md https://www.reddit.com/r/LocalLLaMA/comments/example
```

<Warning>
  Reddit には有効な API キーが必要です。キーがない場合、Reddit の bot 制限により Reddit の URL は失敗します。
</Warning>

### Bridge モード

JS レンダリングやログインが必要なページには Chrome 拡張機能を使用します。

```bash
npx web2md --bridge https://app.example.com/dashboard
```

<Note>
  Bridge モードには、Web2MD Chrome 拡張機能がインストールされ、Chrome が起動している必要があります。CLI は Chrome のネイティブメッセージングプロトコルを介して拡張機能と通信します。
</Note>

### JSON 出力

```bash
npx web2md --json https://example.com/article
```

構造化された出力を返します。

```json
{
  "markdown": "# Article Title\n\nContent here...",
  "metadata": {
    "title": "Article Title",
    "source": "https://example.com/article",
    "wordCount": 1250,
    "tokenCount": 1680,
    "readingTime": 5,
    "date": "2026-04-11T10:30:00.000Z"
  }
}
```

プログラムからの利用や `jq` へのパイプに便利です。

```bash
npx web2md --json https://example.com/article | jq '.metadata.tokenCount'
```

## 最適化済みサイト

Web2MD には、汎用変換よりもクリーンな出力を生成する、以下のサイト向けの組み込みアダプターが含まれています。

<AccordionGroup>
  <Accordion title="最適化対応サイト">
    - **Wikipedia** — クリーンな記事抽出、インフォボックスの処理
    - **arXiv** — 論文の要旨とメタデータ
    - **Hacker News** — コメント付きスレッド
    - **GitHub** — Issue と Pull Request
    - **Stack Overflow** — 質問と回答
    - **dev.to** — ブログ記事
    - **Medium** — 記事(ペイウォールのプレビューを回避)
    - **Substack** — ニュースレター記事
    - **OpenAI Docs** — ドキュメントページ
    - **Mintlify ベースのドキュメント** — Mintlify で構築されたドキュメントサイト
    - **Reddit** — 投稿とコメント(API キーが必要)
  </Accordion>
</AccordionGroup>
