## 概要

Agent Bridge を使うと、AI エージェント(Claude Code、Cursor、Cowork など)が Web2MD Chrome 拡張機能をリモートで操作し、URL をバッチ変換できます — 特に、サーバー側の API アクセスをブロックする Reddit、JS レンダリングされたページ、ログインで保護されたページに有効です。

Web2MD API を呼び出す(Reddit にはブロックされる)代わりに、AI エージェントは[ネイティブメッセージング](https://developer.chrome.com/docs/extensions/develop/concepts/native-messaging)を介してローカルの Chrome 拡張機能にコマンドを送信します。拡張機能は**あなたの実際のブラウザセッション** — cookie やログイン状態を含む — を使ってバックグラウンドタブで各ページを開き、コンテンツを抽出し、Markdown に変換して結果を返します。

<Note>
  Agent Bridge には **PRO プラン**と、マシンにインストールされた**ネイティブメッセージングホスト**が必要です。拡張機能は Chrome 上で開いている必要があります。
</Note>

## アーキテクチャ

```
AI Agent ←MCP/stdio→ MCP Server ←TCP:12315→ Native Host ←NM→ Chrome Extension → Any Website
```

| コンポーネント | 役割 |
|-----------|------|
| **AI Agent** | Claude Code、Cursor、Cowork — MCP ツールを呼び出す |
| **MCP Server** | MCP ツール呼び出しを TCP メッセージに変換 |
| **Native Host** | TCP と Chrome ネイティブメッセージングの間を中継 |
| **Chrome Extension** | タブを開き、HTML を抽出し、Markdown に変換 |

すべての通信は**ローカル**で完結します — 何もマシンの外には出ません。Native Host は `localhost:12315` のみをリッスンします。

## セットアップ

### ステップ 1: MCP Server をビルドする

```bash
cd packages/mcp-server
pnpm build
```

### ステップ 2: ネイティブメッセージングホストをインストールする

```bash
cd packages/mcp-server
./install.sh <your-extension-id>
```

<Tip>
  Developer Mode を有効にした `chrome://extensions` で Extension ID を確認できます。Web2MD を探して ID 文字列をコピーしてください(例: `ijmgpkkfgpijifldbjafjiapehppcbcn`)。
</Tip>

<Warning>
  インストール後は、**Chrome を完全に終了(Mac では Cmd+Q)して再度開く**必要があります。Chrome はネイティブメッセージングのマニフェストを起動時にしか読み込まないため、拡張機能をリロードするだけでは不十分です。
</Warning>

インストールスクリプトは以下を行います。
- ホストファイルを `~/.web2md/` にコピー(`~/Desktop` に対する macOS の TCC 制限を回避)
- `node` の絶対パスを解決(Chrome は最小限の PATH で起動するため)
- Chrome の `NativeMessagingHosts` ディレクトリに NM マニフェストを書き込み

### ステップ 4: 動作確認する

Chrome で `chrome://extensions` を開き、Web2MD の「Service Worker」リンクをクリックしてコンソールを確認します。

```
[Web2MD] Service worker started
[Web2MD] Connected to native host
[Web2MD] Native host TCP relay ready on port 12315
```

この 3 行が表示されれば、Agent Bridge は正常に動作しています。

## 利用可能なツール

Agent Bridge は 3 つの MCP ツールを公開しています — トランスポートとバッチ処理のニーズに応じて選択してください。3 つとも[MCP Server](/ja/advanced/mcp-server)ページで詳しく解説されています。ここでは簡易比較のみを示します。

| ツール | トランスポート | バッチ処理 |
|------|-----------|----------|
| `bridge_convert_url` | HTTP(拡張機能が HTML を取得し、サーバーが変換) | 単一 URL |
| `agent_convert` | ネイティブメッセージング(本ページのアーキテクチャ) | 単一 URL、オプションの `skill` パラメータ |
| `agent_batch_convert` | ネイティブメッセージング | 最大 50 URL、順次処理 |

### agent_convert

ネイティブメッセージングを介して Chrome 拡張機能を使い、単一の URL を変換します。

| パラメータ | 型 | 説明 |
|-----------|------|-------------|
| `url` | string | 変換対象の URL |
| `skill` | string(任意) | Skill の id または名前 — その本文が返却される Markdown に追加されます |

**戻り値:** タイトル・ソース URL・文字数・読了時間を含む Markdown コンテンツ。

**最適な用途:** Reddit スレッド、ログインで保護されたページ、JS レンダリングされたサイト。

### agent_batch_convert

最大 50 件の URL を一括変換します。URL は順次処理されます — 拡張機能が各ページをバックグラウンドタブで開き、コンテンツを抽出し、タブを閉じてから次に進みます。

| パラメータ | 型 | 説明 |
|-----------|------|-------------|
| `urls` | string[] | 変換対象の URL 配列(最大 50) |

**戻り値:** 完了ごとにストリーミングされる URL 単位の結果、およびサマリー。

**最適な用途:** リサーチワークフロー — Reddit スレッド、HN のディスカッション、競合ページを AI 分析用に一括変換。

### bridge_convert_url

ローカルの HTTP ブリッジ経由で拡張機能を通じて(ネイティブメッセージングではなく)単一の URL の HTML を取得し、サーバー側で変換します。単発の変換であれば `agent_convert` と機能的には似ていますが、ネイティブメッセージングホストは不要で、拡張機能がインストールされ実行されていれば十分です。

| パラメータ | 型 | 説明 |
|-----------|------|-------------|
| `url` | string | 変換対象の URL |
| `extensionId` | string(任意) | 拡張機能 ID を上書き(省略時は API キーから自動検出) |

<Info>
  成功したすべての変換は、AI が生成したサマリーとタグとともに、自動的に[ダッシュボードの履歴](https://web2md.org/dashboard/history)に保存されます。
</Info>

## 対応サイト

Agent Bridge は、拡張機能と同じ 27 のサイト別抽出モジュールを使用します — それぞれが、生のページ HTML に頼るのではなく、サイトの DOM や API からクリーンなコンテンツを取り出す方法を把握しています。

<AccordionGroup>
  <Accordion title="ソーシャル & ディスカッション(7)">
    Reddit · Hacker News · Twitter/X · Quora · Xiaohongshu (小红书) · Zhihu (知乎) · Instagram
  </Accordion>
  <Accordion title="AI チャットのエクスポート(2)">
    Claude conversations · その他の AI チャットプラットフォーム(ChatGPT、Gemini など、共通の DOM 抽出モジュール経由)
  </Accordion>
  <Accordion title="ドキュメント & ナレッジベース(6)">
    Wikipedia · arXiv · OpenAI Docs · Mintlify ベースのドキュメントサイト · Notion · Google Docs
  </Accordion>
  <Accordion title="開発 & コード(3)">
    GitHub Issues/PRs · Stack Overflow · dev.to
  </Accordion>
  <Accordion title="パブリッシング(3)">
    Medium · Substack · Product Hunt
  </Accordion>
  <Accordion title="CJK プラットフォーム(4)">
    WeChat (微信公众号) · Bilibili (哔哩哔哩) · Feishu (飞书) — 加えて上記の Xiaohongshu と Zhihu。コード上、このカテゴリ全体が最も差別化度の高い抽出モジュール群と位置づけられています — これらのプラットフォームを扱える競合はほとんどありません。
  </Accordion>
  <Accordion title="動画 & メディア(1)">
    YouTube(文字起こしの抽出)
  </Accordion>
  <Accordion title="仕事 & 生産性(5)">
    Gmail · LinkedIn · Amazon(商品ページ) · Canvas LMS · XenForo フォーラム
  </Accordion>
</AccordionGroup>

専用の抽出モジュールがない URL は、ページ全体の HTML 抽出にフォールバックします — 動作はしますが、専用抽出モジュールが適用するサイト固有のクリーンアップ(ナビゲーション UI、広告、コメントウィジェットなどの除去)は行われません。公開 API を持つサイト(Reddit、Hacker News、GitHub、Stack Overflow)は、可能な場合その API 経由で取得されます — バックグラウンドタブを開くよりも高速で信頼性があります。それ以外はすべて、バックグラウンドタブでページを開きレンダリング後の DOM から抽出します。

## トラブルシューティング

<AccordionGroup>
  <Accordion title="「Native host not installed」または「not found」と表示される">
    Chrome が NM マニフェストを読み込んでいません。**Chrome を完全に終了(Cmd+Q)して再度開いてください。** 拡張機能をリロードするだけでは不十分です。
  </Accordion>
  <Accordion title="「Native host has exited」がすぐに表示される">
    Chrome がホストスクリプトを実行できていません。よくある原因は次の通りです。
    - `node` が見つからない — `node` の絶対パスを使用する `./install.sh` を再実行してください
    - ホストが TCC で保護されたディレクトリ(`~/Desktop`、`~/Documents`)にある — インストールを再実行して `~/.web2md/` に移動してください
  </Accordion>
  <Accordion title="MCP から「Agent connection error」が返される">
    Native Host の TCP サーバーが起動していません。以下を確認してください。
    1. Chrome が開いている
    2. Web2MD 拡張機能が読み込まれている
    3. Service Worker のコンソールに「TCP relay ready on port 12315」と表示されている
  </Accordion>
  <Accordion title="「Failed to extract content from the page」と表示される">
    拡張機能がログインしていません。Chrome で Web2MD のポップアップを開き、PRO アカウントにサインインしてください。
  </Accordion>
  <Accordion title="「Tab load timeout」と表示される">
    対象ページの読み込みに時間がかかりすぎています。低速なサイトでは通常発生します。拡張機能はタブごとに最大 15 秒待機します。Reddit は JSON API 抽出モジュールを使用しタブを必要としないため、Reddit でのタイムアウトは通常、投稿 URL が無効(404)であることを意味します。
  </Accordion>
  <Accordion title="ダッシュボードの履歴に変換結果が表示されない">
    履歴の保存には、PRO アカウントでログイン済みの拡張機能が必要です。保存は fire-and-forget 方式のため、API 呼び出しが静かに失敗すると変換結果は表示されません。認証トークンが有効か確認してください。
  </Accordion>
</AccordionGroup>

## MCP Server との違い

| 機能 | `convert_url` | `bridge_convert_url` | `agent_convert` / `agent_batch_convert` |
|---------|--------------|----------------------|------------------------------------------|
| 実行場所 | サーバー側の API 呼び出し | 拡張機能が HTML を取得し、サーバーが変換 | ローカルの Chrome ブラウザ(ネイティブメッセージング) |
| Reddit 対応 | ❌ Reddit にブロックされる | ✅ 実際のブラウザセッションを使用 | ✅ 実際のブラウザセッションを使用 |
| ログインで保護されたページ | ❌ アクセス不可 | ✅ cookie を使用 | ✅ cookie を使用 |
| JS レンダリングされたページ | ❌ JavaScript 非対応 | ✅ Chrome によるフルレンダリング | ✅ Chrome によるフルレンダリング |
| 必要なセットアップ | API キーのみ | 拡張機能のインストール + 実行 | 拡張機能 + ネイティブメッセージングホストのインストール |
| バッチ処理 | — | 単一 URL のみ | 最大 50 URL(`agent_batch_convert`) |
| Chrome を開いている必要がある | いいえ | はい | はい |

**目安:** 公開ページには `convert_url` を使用してください。拡張機能のブラウザセッションを使いたいがネイティブメッセージングホストはインストールしたくない場合は `bridge_convert_url` を使用してください。Reddit、認証が必要なページ、JS を多用するサイト、バッチ処理が必要な場合、または `agent_convert` の `skill` パラメータで Skill を適用したい場合は `agent_convert` / `agent_batch_convert` を使用してください。
